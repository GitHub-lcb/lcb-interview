# 模拟面试本题脚本进度复盘导出设计

## 背景

本题面试官脚本已经支持逐问状态，用户能看到某个追问是待练、修复中还是已通过。下一步需要把这个状态沉淀成可复制的复盘材料，避免用户练完后只能停留在页面瞬时视图里。

## 目标

- 在本题面试官脚本面板提供“复制进度”能力，导出当前题的追问完成情况。
- 导出内容必须包含脚本阶段、完成比例、下一问建议、每一步状态、验收分和最近练习时间。
- 复盘导出复用本地数据，不依赖付费能力或后端服务，继续保持完全免费。

## 非目标

- 不新增服务端存储。
- 不改变模拟面试评分算法。
- 不引入浏览器自动化校验；本轮遵守 NO web browser 约束，仅做单元测试和构建验证。

## 用户价值

- 用户可以把单题追问训练结果粘贴到笔记、日报或求职复盘里。
- 复盘材料明确下一步动作，不只是展示“练过”。
- 对比只提供题库页面的平台，本项目进一步提供可带走、可追踪、可复盘的免费训练闭环。

## 体验设计

1. 面板顶部原“复制脚本”保留，用于导出面试官提问脚本。
2. 进度条区域新增“复制进度”按钮，用于导出追问完成报告。
3. 复制失败时复用现有 Markdown 下载降级。
4. 导出的 Markdown 标题为“某题 本题面试官脚本进度”，内容包含：
   - 题目信息；
   - 脚本阶段；
   - 脚本进度；
   - 下一步建议；
   - 逐问状态表；
   - 复盘建议。

## 验收标准

- 无追问回答时，导出显示 0/3、待练步骤和下一问建议。
- 已通过某一步时，导出显示已通过、验收分和最近练习日期。
- 未通过某一步时，导出显示修复中，并提醒继续修复。
- 面板点击“复制进度”会写入剪贴板；剪贴板不可用时下载 Markdown。
- `npm run test -- practiceInterviewerScriptProgress PracticeInterviewerScriptPanel` 通过。
- `npm run build` 通过。
