# 模拟面试战报回执验收卡设计

## 背景

战报已经包含训练契约、训练日程、打卡清单和训练回执模板。用户填写回执后，还需要判断这份回执是否足以支撑进入下一轮。如果没有验收卡，用户可能只写了“已完成”，但没有证明目标、证据、阻断项和下一步是否闭环。

## 目标

新增“回执验收卡”，把训练回执模板转成进入下一轮前的检查点：

- 空队列时提示先生成训练回执。
- 非空状态下输出 4 个验收项：目标清晰、证据可查、阻断说明、下一步明确。
- 修复态强调阻断项必须清零；推进态强调证据足够支撑继续加压。
- Markdown 导出同步包含验收卡，方便用户复制到复盘记录。

## 非目标

- 不保存用户填写的回执。
- 不做自然语言解析或自动评分。
- 不新增后端接口。

## 方案

新增纯前端派生模型 `PracticeSessionReceiptAcceptance`。它基于 `buildPracticeSessionTrainingReceipt` 生成验收项，每项包含目标、检查口径、未通过时的动作和入口。状态沿用回执状态：`empty`、`repair`、`advance`。

## 界面

在“训练回执模板”之后、“下一轮训练”之前展示回执验收卡。卡片使用紧凑两列布局，移动端单列。主按钮为“验收训练回执”，跳转到当前训练入口。

## 测试

- Markdown 导出包含“回执验收卡”“目标清晰”“证据可查”“阻断说明”“下一步明确”。
- 空队列 Markdown 包含“等待验收训练回执”。
- 组件测试覆盖回执验收卡面板、四项文案和主按钮跳转。
