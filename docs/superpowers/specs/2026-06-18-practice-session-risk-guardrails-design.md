# 本轮失分禁区设计

## 背景

本轮战报已经能指出证据缺口、复述脚本、验收清单和压力追问，但用户临场重答时仍可能踩同样的表达坑：只讲概念、不讲失败边界、只背标准答案、不说本人动作。这些问题不一定是知识不会，而是表达习惯会在面试官追问下持续扣分。

## 目标

- 在本轮战报新增“本轮失分禁区”。
- 基于压力追问卡生成 3 条“不要这样说 / 改成这样说”的表达护栏。
- 每条护栏包含禁区、风险原因、替代表达和训练入口。
- Markdown 导出与面板展示保持一致。
- 继续前端本地生成，完全免费，不新增后端接口。

## 非目标

- 不做语音实时拦截。
- 不保存用户是否已避开某个禁区。
- 不扩展为通用写作建议，只聚焦本轮模拟面试的重答风险。

## 方案

新增 `PracticeSessionRiskGuardrails` 类型和 `buildPracticeSessionRiskGuardrails(queue, progress)` 构建器。

生成规则：

1. 复用 `buildPracticeSessionPressureProbes`，保证禁区来自当前复述与追问链路。
2. 有压力追问时生成三条护栏：
   - 禁止空讲概念：替换为项目规模、指标和本人动作。
   - 禁止跳过失败边界：替换为异常信号、降级和回滚。
   - 禁止只背标准答案：替换为方案取舍、替代方案和适用边界。
3. 无压力追问时展示等待状态，引导用户先完成模拟面试和复述卡。

面板位置放在“本轮压力追问卡”之后、“下一轮训练”之前。这样用户先看到追问，再看到哪些表达不能出现，最后进入下一轮训练。

## 测试策略

- 工具测试覆盖低分场景导出三条失分禁区。
- 空队列 Markdown 输出等待提示。
- 面板测试覆盖禁区内容和主行动导航。
- 提交前运行聚焦测试、完整前端测试、生产构建和 `git diff --check`。
