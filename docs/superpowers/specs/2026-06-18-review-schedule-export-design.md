# 智能复习队列导出设计

## 背景

学习计划页已经能基于本地学习状态生成“智能复习排期”和“智能复习队列”，包括已逾期、今日到期、即将到期和下次复习时间。这个能力能帮助用户知道下一步该复盘什么，但目前只能停留在页面内。

真正的免费备考闭环需要把复习债带到外部笔记、打卡记录、团队陪练或明日计划里。用户不应该手动抄题、截图或重新整理到期原因。本阶段为智能复习队列增加 Markdown 导出，让复习计划可以一键复制，剪贴板不可用时自动下载。

## 目标

- 导出目标岗位、生成日期、排期概览、复习队列和执行入口。
- 每道复习题包含到期状态、分类、复习次数、下次复习日期、调度原因和题目入口。
- 空队列仍导出明确下一步，提示先标记薄弱题或加入今日计划。
- 页面只负责复制和降级下载，业务内容由 `reviewSchedule` 纯函数生成，避免 UI 与导出内容分叉。
- 增加纯函数和页面交互测试，防止导出出现 `undefined` 或复制按钮失效。

## 非目标

- 不新增后端存储，不上传本地学习状态。
- 不导出完整题目答案，只导出复习排期和入口。
- 不调整现有间隔重复规则，不改变队列排序。

## 内容结构

```markdown
# Java 后端 智能复习队列

生成时间：2026-06-18

## 排期概览
- 已逾期：1 道
- 今日到期：2 道
- 即将到期：3 道
- 下次复习：2026-06-20

## 复习队列
1. HashMap 为什么线程不安全？
   - 状态：已逾期
   - 分类：Java 并发
   - 复习次数：2 次
   - 下次复习：2026-06-16
   - 原因：学习中题 3 天后复习，开始拉开间隔。
   - 入口：/question/1
```

空队列导出：

```markdown
## 复习队列
1. 暂无到期复习题
   - 下一步：先把薄弱题标记为“薄弱”或加入今日计划，再回来生成队列。
   - 入口：/banks
```

## UI 设计

- 在“智能复习队列”标题行右侧增加“复制队列”按钮。
- 使用 `CopyOutlined` 图标，保持小型按钮，不挤压右侧题量。
- 成功复制提示“智能复习队列已复制”。
- 剪贴板不可用时下载 `{岗位}-智能复习队列.md`，并提示“剪贴板不可用，已下载 Markdown 队列”。

## 测试策略

- `reviewSchedule.test.ts` 增加 Markdown 导出测试：
  - 有逾期、今日到期、即将到期题时导出标题、日期、概览、题目、原因和入口。
  - 空队列导出可执行下一步。
  - 内容不出现 `undefined`。
- `StudyPlan/index.test.tsx` 增加页面交互测试：
  - 写入本地学习状态。
  - mock `navigator.clipboard.writeText`。
  - 点击“复制队列”后断言写入 Markdown 包含标题、概览和题目。
