# 冲刺报告评分影响说明设计

## 背景

今日闭环验收已经能解释当天模拟面试评分如何改变今日计划，但总冲刺报告仍只展示闭环状态、完成率和待办。如果用户复制的是总报告，低分题为什么重新进入补弱、高分题为什么计入完成仍会丢失上下文。

## 目标

- 冲刺报告的“今日计划闭环”章节展示评分影响。
- 复用 `DailyPlanCompletion.statusImpacts`，不重复计算分数状态规则。
- 有评分影响时列出题目、分数和计划变化说明。
- 无评分影响时保留空状态说明，提醒用户完成计划内模拟评分后报告会解释计划变化。
- 不改页面组件，不新增类型。

## 输出示例

```markdown
## 今日计划闭环
- 状态：今日闭环还有风险
- 完成率：0%
- 评分影响：Java 并发 题目 1，60 分，已自动标记薄弱，并留在今日计划继续补强。
```

## 验收

- 冲刺报告包含计划内当天评分影响。
- 计划外评分不进入冲刺报告闭环影响。
- 无评分影响时报告仍可读，不出现 `undefined`。
- `npm run test -- sprintReport`、`npm run test` 和 `npm run build` 通过。
