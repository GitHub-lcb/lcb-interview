# 冲刺报告高分素材导出设计

## 背景

高分表达素材库已经能在学习计划页把 80 分以上的模拟回答沉淀成可复述话术，但冲刺报告导出仍然没有包含这些素材。用户把报告复制到外部文档、聊天窗口或个人知识库时，仍需要手动回到页面翻找高分片段。

## 目标

- 在 `buildSprintReportMarkdown` 中新增“高分表达素材库”章节。
- 复用 `buildInterviewMaterialVault(progress)`，不重复实现素材筛选和抽取逻辑。
- 导出素材库标题、摘要、指标和最多 6 条素材片段。
- 空素材库也输出行动提示，避免导出的报告出现信息断层。
- 保持完全前端、本地计算、免费可用，不新增接口和存储字段。

## 方案

在 `sprintReport.ts` 中新增：

1. `materialVault = buildInterviewMaterialVault(progress)`。
2. `renderMaterialVaultSection(materialVault)`。
3. 在最后 24 小时简报之后、备考健康度之前插入该章节。
4. 在“下一步行动”章节补充素材库主行动。

章节格式：

```md
## 高分表达素材库
- 状态：高分表达素材已可复用
- 摘要：已沉淀 3 条高分素材...
- 高分样本：3，80 分以上才沉淀
- 覆盖分类：2，素材分布稳定
- 平均素材分：89 分，可继续打磨
- 可复述素材：3，面试前直接复盘
- 项目场景：如何设计秒杀库存扣减？（88 分）项目场景是...，原因...，/question/1
```

## 测试

- 扩展 `sprintReport.test.ts` 的非空报告用例，加入 80+ 分回答并断言导出“高分表达素材库”、素材标题、素材片段和下一步素材行动。
- 扩展空报告用例，断言空素材库提示和 `/practice` 行动入口。
- 复跑前端全量测试、生产构建、后端测试和 `git diff --check`。
