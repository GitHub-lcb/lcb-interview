# 答案差距校准设计

## 背景

练习页已经支持模拟面试评分和追问加压训练，题目详情页也能评估标准答案质量。但用户在练习页提交回答后，还缺少一层关键反馈：自己的回答和标准答案之间到底差在哪里。只有知道漏掉了哪些标准模块，用户才能把追问训练沉淀成可复述答案。

## 目标

- 在练习页评分后，对比用户回答与题目标准答案模块。
- 输出覆盖模块、缺失模块、每个模块的补强方向和重写提纲。
- 完全基于本地题目详情和用户回答，不新增后端接口、不依赖 AI。
- 与现有评分、追问加压训练形成闭环：先评分，再追问，再按标准答案校准。

## 设计方案

新增 `buildAnswerGapReport(question, answer)` 工具函数：

- 从题目详情读取模块：30 秒口径、标准回答、原理深挖、对比分析、适用场景、风险误区、项目落地。
- 对每个模块提取关键词，计算用户回答覆盖度。
- 输出模块状态：
  - `covered`：覆盖较充分。
  - `partial`：有触及但不完整。
  - `missing`：几乎没有覆盖。
- 聚合为 0-100 的差距校准分。

新增 `AnswerGapPanel`：

- 在练习页评分结果后展示。
- 仅当当前题目详情可用时出现。
- 展示总分、缺失模块、补强建议和重写提纲。

## 行为规则

- 用户回答为空时直接返回 0 分，并提示先完成基础回答。
- 题目某个标准模块为空时不纳入评分，避免惩罚题库本身暂缺内容。
- 风险误区和项目落地权重更高，因为它们最能拉开面试表达质量。
- 重写提纲最多 5 条，优先取缺失模块，避免面板过长。

## 测试策略

- 单元测试覆盖空回答、缺失风险/项目模块、完整回答高分、Markdown 清理和空标准模块不扣分。
- 前端全量测试和构建验证练习页接入。
