# 答案差距校准导出设计

## 背景

练习页已经能在用户提交回答后展示“答案差距校准”，把用户回答和标准答案拆成 30 秒口径、标准回答、原理、对比、场景、风险和项目落地等模块。但这个校准结果目前只能在页面内查看，用户无法把缺口、证据和重写提纲复制到笔记中复盘。

本阶段要把答案差距校准导出为 Markdown，让用户在完全免费的前提下，把单题回答差距保存、分享和持续打磨。

## 目标

- 新增 `buildAnswerGapMarkdown(question, answer, now)`。
- Markdown 包含题目、分类、难度、校准分、摘要、模块明细、缺失模块、已覆盖模块和重写提纲。
- 在 `AnswerGapPanel` 增加“复制校准”按钮。
- 剪贴板不可用时下载 Markdown，避免浏览器权限限制导致导出失败。
- 增加组件测试，覆盖按钮存在和剪贴板内容。

## 非目标

- 不修改答案差距评分算法。
- 不新增后端接口。
- 不保存云端历史版本。
- 不把标准答案全文导出到额外文件，只导出差距分析和提纲。

## 方案

### 方案 A：只复制页面可见文本

实现快，但无法保证 Markdown 结构稳定，也无法覆盖空回答和缺失模块列表。

### 方案 B：新增 Markdown 构建函数并由面板调用

复用 `buildAnswerGapReport`，导出稳定结构。纯函数可单测，组件只负责复制和降级下载。

### 方案 C：把答案差距并入本轮战报导出

复盘链路更完整，但需要把当前题、回答草稿和本轮队列组合在一起，范围更大，适合后续迭代。

选择方案 B。它能把单题差距校准变成可携带产物，并保持实现边界清晰。

## Markdown 结构

```md
# HashMap 为什么线程不安全？ 答案差距校准

生成时间：2026-06-17
分类：Java 集合
难度：HARD

## 校准摘要
- 分数：42
- 状态：回答和标准答案差距较大
- 说明：当前差距校准分 42，优先补齐 风险误区、项目落地。

## 模块明细
- 30 秒口径：65 分，partial，已覆盖：HashMap、多线程；先用一句话给结论，再补核心原因。
- 风险误区：0 分，missing，回答中暂未命中该模块关键点；补适用边界、失败场景和常见误区。

## 优先补齐
- 风险误区：补适用边界、失败场景和常见误区。
- 项目落地：补项目实践、监控指标、压测或迁移方案。

## 重写提纲
1. 先补「风险误区」：补适用边界、失败场景和常见误区。
2. 先补「项目落地」：补项目实践、监控指标、压测或迁移方案。
```

空回答也导出：

```md
## 校准摘要
- 分数：0
- 状态：先完成基础回答
- 说明：回答为空，暂时无法和标准答案校准。
```
