# 面试错因修复计划导出设计

## 背景

学习计划页已经能展示面试错因本和三步修复计划。用户在临近面试时通常需要把计划复制到笔记、发给同伴复盘，或离线保存。现有冲刺报告已经提供复制 Markdown 的体验，错因修复计划也应提供同等能力。

## 目标

- 将当前面试错因本和修复计划导出为 Markdown。
- 优先复制到剪贴板，失败时自动下载 `.md` 文件。
- Markdown 中包含生成日期、目标岗位、错因摘要、修复步骤和首要行动。
- 空状态也要导出可执行的下一步，不输出空文件。
- 完全本地运行，不新增后端接口。

## 设计方案

新增 `buildInterviewRecoveryMarkdown(ledger, plan, targetRole, now)`：

- 标题：`<目标岗位> 面试错因修复计划`
- 基本信息：生成日期、计划总耗时、当前模式
- 错因概览：错因本标题、摘要、首要行动
- 修复步骤：逐条列出标题、耗时、原因、入口

在 `InterviewMistakeLedgerPanel` 的修复计划头部增加“复制计划”按钮：

- 调用 Markdown 生成器。
- 优先 `navigator.clipboard.writeText`。
- 失败时创建 Blob 并下载。
- 文件名使用目标岗位清洗后的安全名称。

## 行为规则

- 不把完整题目内容写入 Markdown，只输出题目 ID 和训练入口，避免文件过长。
- 不输出 `undefined`、空 URL 或空标题。
- 剪贴板失败要给出可恢复反馈，并自动下载。

## 测试策略

- 单元测试 Markdown 生成器：高风险错因、空状态。
- 断言包含标题、日期、步骤、首要行动，不包含 `undefined`。
- 前端全量测试和构建验证组件接入。
