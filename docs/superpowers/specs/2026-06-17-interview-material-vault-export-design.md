# 高分表达素材库导出设计

## 背景

高分表达素材库已经能把 80 分以上的模拟面试回答沉淀成结论话术、项目场景和风险边界，但当前面板只能在页面内查看。用户面试前需要把这些高分片段复制到个人笔记、简历项目话术或最后冲刺清单时，仍要手动逐条整理。

本阶段要把素材库导出为 Markdown，让用户在完全免费的前提下，把已经验证过的高分表达保存成可复述素材包。

## 目标

- 新增 `buildInterviewMaterialVaultMarkdown(progress, now)`。
- Markdown 包含目标岗位、生成日期、素材库摘要、指标、素材清单和下一步动作。
- 在 `InterviewMaterialVaultPanel` 增加“复制素材”按钮。
- 剪贴板不可用时下载 Markdown，避免权限限制导致导出失败。
- 增加纯函数和组件测试，覆盖可复用素材库与空素材库。

## 非目标

- 不调整素材抽取阈值，仍然只沉淀 80 分及以上回答。
- 不新增后端接口。
- 不新增编辑、删除或云端收藏功能。
- 不把完整历史回答导出，只导出已抽取的高分片段和沉淀原因。

## 方案

### 方案 A：复制素材卡片可见文本

实现快，但结构不稳定，也无法导出指标、空状态和下一步动作。

### 方案 B：新增 Markdown 构建函数并由面板调用

复用 `buildInterviewMaterialVault(progress)`，输出稳定 Markdown。纯函数可单测，面板只负责复制和降级下载。

### 方案 C：只依赖冲刺报告里的素材库章节

冲刺报告已经包含素材库，但用户在素材库面板里看到高分片段时仍需要即时复制；只依赖总报告会增加操作路径。

选择方案 B。它把素材库从“页面卡片”升级成可携带的面试素材包，同时保持当前数据模型不变。

## Markdown 结构

```md
# Java 后端 高分表达素材库

生成时间：2026-06-17

## 素材概览
- 状态：高分表达素材已可复用
- 摘要：已沉淀 2 条高分素材，覆盖 2 个分类，平均 88 分。
- 下一步：复盘高分素材，/study

## 指标
- 高分样本：2，80 分以上才沉淀
- 覆盖分类：2，素材分布稳定

## 素材清单
1. 如何设计秒杀库存扣减？
   - 分类：系统设计
   - 类型：项目场景
   - 得分：88 分
   - 片段：项目场景是订单高峰期并发扣库存...
   - 沉淀原因：88 分回答里有可迁移项目场景...
   - 入口：/question/1
```

空素材库也导出可执行提示：

```md
## 素材清单
- 暂无高分素材。先完成一次 80 分以上模拟回答，再沉淀可复述话术。
```
