# 备考健康雷达设计

## 背景

当前首页已经具备备考就绪度、今日冲刺任务、模拟面试复盘、岗位能力地图和智能复习排期。它们分别解决“下一步做什么”和“某个维度如何补强”，但用户仍缺少一个更高层的免费诊断入口：一眼判断当前备考是否健康、风险来自哪里、为什么系统给出这个判断。

## 目标

- 将本地学习进度、复习排期、岗位能力地图和模拟面试复盘合成为一个 0-100 的备考健康度。
- 用 4 个维度展示风险：复习债、能力覆盖、表达稳定性、执行节奏。
- 为最高风险维度给出一条可执行建议，直接跳转到对应训练入口。
- 全部基于浏览器本地数据计算，不增加登录、付费、后端接口或外部服务依赖。

## 设计取舍

### 推荐方案：本地聚合诊断

复用已有工具函数构建诊断层：

- `buildScheduledReviewQueue` 和 `summarizeReviewSchedule` 提供复习债信号。
- `buildAbilityMap` 提供岗位能力覆盖信号。
- `buildInterviewReviewSummary` 提供表达稳定性信号。
- `StudyProgress.dailyPlan` 和已跟踪题目数提供执行节奏信号。

这种方式可以用最小改动放大已有能力，避免重复算法，也不会把首页变成依赖后端成功返回才能工作的页面。

### 不采用：新增后端健康报告接口

后端聚合能做跨端同步，但当前核心卖点是完全免费和本地可用。此阶段新增接口会引入数据库、鉴权和同步边界，收益不如先把本地闭环打磨完整。

## 数据模型

新增类型：

- `PrepHealthDimension`：单个健康维度，包含 key、label、score、status、description、detail。
- `PrepHealthAction`：最高优先级行动，包含 label、description、to。
- `PrepHealthReport`：整体健康度、等级、标题、摘要、维度列表和主行动。

健康等级：

- `empty`：几乎没有本地进度，需要先建立学习轨迹。
- `risk`：健康度低于 50，存在明显风险。
- `watch`：健康度 50-79，有短板但可控。
- `healthy`：健康度 80 及以上，适合持续冲刺。

## 评分规则

- 复习债：逾期越多扣分越重，今日到期轻度扣分，最低 0 分。
- 能力覆盖：取最值得处理的岗位能力项；无轨迹时为 0，准备度高则得分高。
- 表达稳定性：无模拟记录时为 35 分；下降趋势和低均分降低分数。
- 执行节奏：今日计划为空扣分，已有计划和已跟踪题目提升分数。

整体分取四个维度平均分。主行动选择最低分维度，确保建议聚焦最大风险。

## UI 方案

新增 `PrepHealthRadarPanel`，放在首页 `DailyMissionPanel` 之后、细分复盘面板之前。

展示结构：

- 左侧或顶部：健康度大分、等级标题、摘要、主行动按钮。
- 右侧或下方：四个维度卡片，展示分数、状态、描述和诊断依据。

样式延续现有 Ant Design + 自定义 CSS 的 8px 卡片、低饱和背景和多色风险提示，不引入新 UI 库。

## 测试策略

- 工具函数测试覆盖空进度、复习逾期、能力短板、面试下降、今日计划为空和维度排序。
- 组件本身保持轻展示，不新增复杂交互测试；构建和现有单元测试验证集成。
