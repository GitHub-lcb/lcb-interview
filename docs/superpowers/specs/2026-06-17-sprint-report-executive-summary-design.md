# 冲刺报告一页作战摘要设计

## 背景

冲刺报告已经包含健康雷达、四维诊断、今日计划闭环、今日作战简报、面试表达简报、错题账本和恢复计划。信息密度已经接近完整私人教练复盘，但复制出去后第一屏仍然需要向下滚动才能看出“现在最该做什么”。

面试备考用户通常在碎片时间复盘：通勤、睡前、面试前 10 分钟。他们需要先看到总分、最大风险、今日闭环、错题恢复和优先动作，再决定是否看详细章节。因此报告顶部需要一个“一页作战摘要”，把长报告变成先决策、再展开的结构。

## 目标

- 在 `buildSprintReportMarkdown` 的基础信息后、详细章节前新增 `## 一页作战摘要`。
- 摘要复用已生成的 `PrepHealthReport`、`DailyPlanCompletion`、`InterviewMistakeLedger` 和 `InterviewRecoveryPlan`。
- 输出总分、最大风险、今日闭环状态、今日完成率、错题账本状态、恢复计划标题和三条优先动作。
- 空数据也必须可行动，不能出现空章节或 `undefined`。
- 保持纯前端、本地计算、完全免费，不新增接口。

## 非目标

- 不新增单独的摘要工具函数文件；报告摘要只服务于 Markdown 导出。
- 不改变页面 UI 排版和复制按钮交互。
- 不重新排序详细章节。
- 不把摘要做成复杂评分模型，避免与健康雷达重复。

## 设计方案

新增 `renderExecutiveSummarySection(health, completion, ledger, recoveryPlan)`：

```text
## 一页作战摘要
- 总分：56，最大风险：复习债
- 今日闭环：今日闭环还有风险，完成率 0%
- 错题恢复：面试错因本，三步修复首要错因
- 先做健康动作：处理复习债 - ...
- 再做今日动作：先清复习债 - ...
- 最后做恢复动作：回炉训练 - ...
```

摘要放在报告基础信息之后、`备考健康度` 之前。这样复制后的报告第一屏先给结论，下面章节再解释原因。

## 行为规则

- `targetRole` 和日期仍由原函数处理，摘要不重复输出标题和生成时间。
- 摘要动作使用现有 `primaryAction`，保留入口路径。
- 错题恢复展示 `ledger.title` 和 `recoveryPlan.title`，空账本时自然呈现“先建立样本”。
- 如果动作路径为空，恢复动作降级为 `/practice`。

## 测试策略

- 扩展 `sprintReport.test.ts`：
  - 非空报告断言包含 `## 一页作战摘要`、`总分：`、`今日闭环：今日闭环还有风险`、`错题恢复：` 和恢复入口。
  - 空报告断言摘要包含 `今日计划待验收`、`先建立面试样本`，且不出现 `undefined`。
- RED 阶段运行 `npm run test -- sprintReport`，确认测试因缺少摘要失败。
- GREEN 阶段实现最小渲染函数并接入报告。
- 提交前运行前端全量测试、前端构建、后端测试、`git diff --check` 和 HTTP 200 探测。
