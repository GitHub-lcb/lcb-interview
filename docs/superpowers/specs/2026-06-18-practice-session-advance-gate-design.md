# 模拟面试战报下一轮准入闸门设计

## 背景

战报已经能生成下一轮训练契约、训练日程、打卡清单、训练回执模板和回执验收卡。当前还缺少一个明确的“能否进入下一轮”裁决层。没有准入闸门时，用户虽然能看到下一轮训练建议，但不知道自己应该立刻推进，还是先把本轮回执和阻断项修到位。

## 目标

新增“下一轮准入闸门”，把回执验收结果转成清晰裁决：

- 空队列时提示先生成回执验收样本。
- 修复态显示“暂缓进入下一轮”，列出需要先修复的准入条件。
- 推进态显示“允许进入下一轮”，列出已经满足的准入条件。
- Markdown 导出同步包含准入闸门，方便复制到复盘记录。
- 主按钮根据状态跳回修复入口或进入下一轮训练入口。

## 非目标

- 不保存用户真实填写的回执内容。
- 不新增后端接口。
- 不做自然语言自动解析。
- 不替代下一轮训练队列排序。

## 方案

新增纯前端派生模型 `PracticeSessionAdvanceGate`。它基于 `buildPracticeSessionReceiptAcceptance` 生成准入项，并复用当前战报主行动入口。状态分为：

- `empty`：等待建立验收样本。
- `blocked`：本轮仍需修复，不建议进入下一轮。
- `ready`：验收信号足够，允许进入下一轮训练。

准入项包含条件、当前状态、行动和入口。修复态优先提示目标、证据、阻断、下一步四类缺口；推进态显示四项均已满足，帮助用户放心进入下一轮。

## 界面

在“回执验收卡”之后、“下一轮训练”之前展示准入闸门。卡片使用紧凑列表，修复态使用警示色，推进态使用通过色，空态使用中性色。移动端保持单列，避免文本拥挤。

## 测试

- Markdown 导出包含“下一轮准入闸门”“暂缓进入下一轮”或“允许进入下一轮”。
- 空队列 Markdown 包含“等待建立准入样本”。
- 组件测试覆盖准入闸门面板、准入条件文案和主按钮跳转。
