# LCB Interview 备考指挥中心设计

## 背景

上一期已经完成“完全免费、无会员墙”的公开定位，并新增了备考路线和面试场景题单。下一步要把“比 mianshiya.com 强很多”落到用户每天打开网站后的核心体验：系统不能只展示题目入口，而要告诉用户现在离面试可用状态还有多远、最大短板是什么、下一步应该做什么。

本期不做账号系统、社区、移动端、小程序或 IDE 插件。先把现有本地学习进度、今日计划、模拟面试记录和题目快照组织成一个“备考指挥中心”。

## 目标

首页新增一个可解释的备考指挥中心，给用户 3 类信息：

1. **备考就绪度**：用掌握题、薄弱题、学习中题、模拟面试次数、今日计划等本地数据计算 0-100 分。
2. **最大短板**：用规则判断当前最阻碍面试准备的因素，例如没有开始跟踪、薄弱题过多、缺少模拟面试、今日计划为空。
3. **下一步动作**：生成 3 个可点击动作，直接跳到学习计划、模拟面试、题库或路线页。

## 用户价值

- 新用户不再面对空白题库，而是看到明确的启动动作。
- 已学习用户不再靠记忆决定复习顺序，而是看到短板和下一步。
- 站点差异从“题库免费”升级为“免费智能备考系统”。

## 产品设计

### 1. 就绪度卡片

展示：

- 当前分数，例如 `72`
- 等级文案，例如“进入面试表达强化期”
- 简短解释，例如“掌握题稳定增长，但模拟面试样本偏少”
- 4 个影响因子：掌握、薄弱、模拟、计划

评分规则必须可解释，不依赖 AI：

- 有跟踪题、掌握题、模拟面试和今日计划会加分。
- 薄弱题比例过高会扣分。
- 完全新用户也给启动建议，而不是显示失败。

### 2. 最大短板

短板优先级：

1. 没有跟踪题：先浏览题库并加入计划。
2. 今日计划为空：先生成计划。
3. 薄弱题数量多于掌握题：先复盘薄弱题。
4. 没有模拟面试记录：先做一轮表达训练。
5. 否则进入持续强化：继续推进路线和面试场景。

### 3. 下一步动作

动作包含：

- `开始模拟面试`
- `生成/打开今日计划`
- `查看备考路线`
- `继续刷题`

每个动作直接使用现有路由，不新增后端接口。

## 技术设计

### 新增纯函数模块

新增 `frontend/src/utils/studyStrategy.ts`：

- `buildStudyStrategy(progress: StudyProgress): StudyStrategy`
- 聚合 `questionStates`、`dailyPlan`、`interviewAttempts`
- 输出就绪度、短板、动作和影响因子

复杂评分逻辑写中文注释，解释权重设计原因。

### 新增组件

新增 `frontend/src/components/StudyCommandCenter.tsx`：

- 读取 `useStudyProgress()`
- 调用 `buildStudyStrategy`
- 渲染就绪度、短板、影响因子和动作按钮
- 使用现有 Ant Design `Button`、`Progress`

### 首页集成

修改 `frontend/src/pages/Home/index.tsx`：

- 将 `StudyCommandCenter` 放在 `StudyDashboard` 下方、免费承诺区上方。
- 保持现有学习面板和题库入口不变。

### 类型

在 `frontend/src/types.ts` 增加：

- `StudyStrategy`
- `StudyStrategyFactor`
- `StudyStrategyAction`

## 测试策略

新增 `frontend/src/utils/studyStrategy.test.ts`：

- 新用户得到低分和启动动作。
- 薄弱题多时最大短板指向薄弱复盘。
- 有模拟面试、掌握题和计划时分数提升。
- 所有动作必须指向已有路由。

继续运行：

- `npm run test -- studyStrategy`
- `npm run test`
- `npm run build`
- `mvn test`

## 验收标准

- 首页出现备考指挥中心。
- 新用户、薄弱题用户、已有训练用户看到不同建议。
- 分数和建议完全免费、无需登录、无需后端。
- 评分逻辑有中文注释说明权重原因。
- 所有新增提交使用中文提交信息。
