# 单题评分闭环导出设计

## 背景

模拟面试已经能在单题评分后给出“面试后闭环教练”，把得分、最低维度、答案长度、追问数和下一步动作展示给用户。但这些闭环建议目前只停留在当前页面，用户不能把本题的评分证据、行动清单和重答 Prompt 保存到自己的复盘材料中。

本阶段要把单题评分闭环导出为 Markdown，让用户在完全免费的前提下，把每道题的评分后动作沉淀为可复制、可归档、可二次打磨的复盘资产。

## 目标

- 新增 `buildPracticeFeedbackClosureMarkdown(question, answer, feedback, now)`。
- Markdown 包含题目、生成日期、分类、难度、闭环摘要、评分维度、闭环指标、行动清单和原始回答。
- 在 `PracticeFeedbackClosurePanel` 增加“复制闭环”按钮。
- 剪贴板不可用时下载 Markdown，避免权限或浏览器限制导致导出失败。
- 增加纯函数和组件测试，覆盖低分修复场景、短回答场景和剪贴板内容。

## 非目标

- 不修改现有评分算法。
- 不新增后端接口。
- 不保存云端历史版本。
- 不调整模拟面试队列生成逻辑。

## 方案

### 方案 A：复制面板可见文本

实现最快，但会丢失评分维度明细、Prompt 和原始回答，也难以用单测保证结构稳定。

### 方案 B：新增 Markdown 构建函数并由面板调用

复用 `buildPracticeFeedbackClosure` 的闭环结果，导出稳定结构。纯函数单测覆盖内容，面板只负责复制和降级下载。

### 方案 C：把单题闭环并入本轮模拟面试战报

战报链路更完整，但当前用户的即时需求是把“刚评分的这一题”带走；并入整轮战报会扩大状态依赖，适合后续迭代。

选择方案 B。它能快速补上单题闭环产物，保持实现边界清晰，并与现有答案差距校准导出保持一致。

## Markdown 结构

```md
# HashMap 为什么线程不安全？ 单题评分闭环

生成时间：2026-06-17
分类：Java 并发
难度：MEDIUM

## 闭环摘要
- 本轮得分：52 分
- 闭环等级：repair
- 闭环判断：先修复最低分维度
- 下一步：重答结构化
- 说明：当前最需要修复「结构化」，不要直接跳到下一题。

## 评分维度
- 覆盖度：62 分，覆盖一般
- 结构化：42 分，结构不足

## 闭环指标
- 本轮得分：52 分，需要修复
- 最低维度：结构化 42，结构不足

## 行动清单
1. 重答结构化：最低维度是结构化，先重答这一块再继续下一题。
   Prompt：请重新回答「HashMap 为什么线程不安全？」，重点补齐结构化。
2. 标记薄弱：把这道题放入复习债，后续学习计划会优先安排回炉。

## 原始回答
HashMap 在多线程下 put 和 resize 都可能出现覆盖、丢数据和结构异常。
```

短回答也导出可执行动作：

```md
## 行动清单
1. 补场景后重答：答案太短，先补一个项目场景、失败边界和验证指标，再重新提交评分。
   Prompt：请重新回答「HashMap 为什么线程不安全？」...
```
