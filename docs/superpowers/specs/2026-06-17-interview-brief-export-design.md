# 面试前冲刺简报导出设计

## 背景

面试前冲刺简报已经能把学习路线、掌握题、薄弱风险、复习队列和模拟面试状态汇总成三个区块：可主动表达、必须规避、开口热身。它是学习计划页里最常态的备考总览，适合每天开始训练前确认方向。

当前简报只能在页面上查看，用户无法把优势素材、风险提醒和热身题队列复制到自己的复习文档。本阶段为它增加 Markdown 导出，补齐“常态备考简报 -> 最后 24 小时简报 -> 30 分钟急救包”的免费可携带链路。

## 目标

- 新增 `buildInterviewBriefMarkdown(routes, progress, now)` 纯函数，复用现有 `buildInterviewBrief(routes, progress, now)`。
- Markdown 包含目标岗位、生成日期、简报概览、下一步、可主动表达、必须规避、开口热身。
- 在 `InterviewBriefPanel` 增加“复制简报”按钮。
- 剪贴板不可用时下载 Markdown 文件，保证用户能带走内容。
- 增加工具函数和组件交互测试，覆盖风险简报和空状态导出。

## 非目标

- 不修改面试简报的风险判断、优势排序和热身队列算法。
- 不新增后端接口、账号、付费能力或云端存档。
- 不把完整标准答案导出，只导出本地学习轨迹生成的行动简报。
- 不使用浏览器 QA；继续使用命令行测试、构建和 HTTP 探测。

## 方案

### 方案 A：复制页面三列可见文本

实现快，但缺少生成日期、主行动入口和空状态稳定结构，也很难单测。

### 方案 B：新增 Markdown 构建纯函数并由面板调用

复用 `buildInterviewBrief` 结构化输出。导出函数负责 Markdown，面板负责复制和下载降级。测试可以覆盖空状态、风险状态和入口字段。

### 方案 C：只依赖冲刺总报告

冲刺总报告已含面试简报信息，但常态训练时用户需要更轻的一页总览。只依赖总报告会让“每天带走一份简报”的路径过重。

选择方案 B。它把面试前冲刺简报变成独立可携带的免费备考资料，同时保持现有计算边界稳定。

## Markdown 结构

```md
# Java 后端工程师 面试前冲刺简报

生成时间：2026-06-17

## 简报概览
- 状态：Java 后端工程师 面试前先控风险
- 摘要：已跟踪 2 道题，沉淀 1 个优势方向...
- 下一步：先处理风险，/study
- 说明：已有 1 道题错过复习窗口...

## 可主动表达
1. MySQL 可以主动表达
   - 指标：1 掌握
   - 说明：可从「MySQL 题目 2」切入...
   - 入口：无

## 必须规避
1. 复习逾期会拖累临场稳定性
   - 指标：1 逾期
   - 说明：已有 1 道题错过复习窗口...
   - 入口：/study

## 开口热身
1. Java 并发题目 1
   - 指标：逾期复习
   - 说明：Java 并发 · MEDIUM
   - 入口：/question/1
```

空状态也必须导出明确下一步：

```md
## 必须规避
1. 还没有学习轨迹
   - 入口：/banks
```

## UI

- `InterviewBriefPanel` 头部主行动旁增加“复制简报”小按钮。
- 使用 `CopyOutlined` 图标，保持和其他导出按钮一致。
- 成功复制提示“冲刺简报已复制”。
- 剪贴板失败时下载 `{目标岗位}-面试前冲刺简报.md`。
- 用一个头部按钮容器承载复制按钮和主按钮，避免按钮挤压标题文本。
