# 模拟面试战报训练日程打卡清单设计

## 背景

“下一轮训练日程”已经把训练契约拆成预热、限时作答和验收复盘三个时间块，但用户完成每个时间块后还需要知道应该留下什么证据。没有打卡口径时，用户容易把“看过日程”误当成“完成训练”。

## 目标

在战报中新增“训练日程打卡清单”，把每个时间块转成可核销的完成证据：

- 空队列时提示先生成训练日程。
- 非空日程按时间块输出 3 条打卡项。
- 每条打卡项包含完成口径、证据模板和复盘问题。
- Markdown 导出同步包含打卡清单，方便复制到笔记或待办工具。

## 非目标

- 不在本轮新增持久化打卡状态。
- 不新增后端接口。
- 不引入日历、提醒或通知能力。

## 方案

新增纯前端派生模型 `PracticeSessionScheduleChecklist`。它基于 `buildPracticeSessionTrainingSchedule` 输出的日程项生成清单，每个清单项绑定对应时间块，要求用户留下“已完成什么、证据是什么、下一步是否继续”的文字证据。

打卡清单状态复用日程状态：

- `empty`：等待训练日程。
- `repair`：按修复日程核销。
- `advance`：按推进日程核销。

## 界面

在训练日程之后、下一轮训练题组之前展示打卡清单。每条卡片展示阶段、完成口径、证据模板和复盘问题。主按钮沿用日程入口，名称为“按清单打卡”。

## 测试

- 工具测试覆盖 Markdown 中的“训练日程打卡清单”“完成口径”“证据模板”“复盘问题”。
- 空队列 Markdown 必须提示“等待生成打卡清单”。
- 组件测试覆盖打卡清单面板、三项文本和按钮跳转。
