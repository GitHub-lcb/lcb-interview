# 面试追问防线导出设计

## 背景

面试追问防线已经能聚合最近模拟面试里最容易被连续追问的问题，展示题目、追问、压力点和回答引导。但当前结果只能在学习页查看，用户无法把这份防线清单复制到面试前笔记或训练脚本里。

本阶段要把追问防线导出为 Markdown，让用户在完全免费的前提下，把“今天最该防守的追问”保存成可离线训练的清单。

## 目标

- 新增 `buildInterviewFollowUpDefenseMarkdown(progress, now)`。
- Markdown 包含目标岗位、生成日期、摘要、指标、追问清单和下一步动作。
- 在 `InterviewFollowUpDefensePanel` 增加“复制防线”按钮。
- 剪贴板不可用时下载 Markdown，避免权限限制导致导出失败。
- 增加纯函数和组件测试，覆盖高风险追问和空状态。

## 非目标

- 不修改追问优先级算法。
- 不新增后端接口。
- 不保存云端历史版本。
- 不把完整模拟回答导出，只导出追问、防守压力点和回答引导。

## 方案

### 方案 A：复制面板可见文本

实现快，但会丢失指标、入口和空状态行动，也难以用单测保证结构稳定。

### 方案 B：新增 Markdown 构建函数并由面板调用

复用 `buildInterviewFollowUpDefense(progress)`，输出稳定结构。纯函数单测覆盖内容，面板只负责复制和降级下载。

### 方案 C：只依赖冲刺报告里的追问防线章节

冲刺报告已经包含追问防线，但用户在面板里看到清单时仍需要即时复制；只依赖总报告会增加操作路径。

选择方案 B。它把追问防线从“页面提醒”升级为可携带训练清单，并保持当前算法边界不变。

## Markdown 结构

```md
# Java 后端 面试追问防线

生成时间：2026-06-17

## 防线概览
- 状态：先补高风险追问
- 摘要：发现 1 道低分回答，先处理前 5 条高风险追问...
- 下一步：先修追问短板，/practice?queue=1

## 指标
- 防线追问：5，今日优先演练
- 最近均分：62 分，仍需加压

## 追问清单
1. HashMap 为什么线程不安全？
   - 分类：Java 并发
   - 维度：场景细节
   - 得分：62 分
   - 追问：如果面试官追问线上场景，你会怎么验证？
   - 压力点：面试官在追项目场景...
   - 回答引导：用一个真实场景回答...
   - 入口：/practice?queue=1
```

空状态也导出行动提示：

```md
## 追问清单
- 暂无追问防线。先完成一次模拟面试，系统会生成可防守的追问清单。
```
