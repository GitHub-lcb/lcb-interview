# 本轮薄弱能力雷达设计

## 背景

练习战报已经能展示本轮材料库、追问防线、脚本总控、错因账本和错因验收，但用户仍需要自己从多道题的维度分数里判断“到底哪个面试能力最拖后腿”。这个判断如果只靠平均分，会掩盖覆盖度、结构化、场景细节、风险意识之间的差异。

## 目标

- 在本轮战报中新增“本轮薄弱能力雷达”，只基于当前练习队列的最新模拟面试记录。
- 聚合四个稳定维度：覆盖度、结构化、场景细节、风险意识。
- 展示最低维度、平均分、影响题数、受影响题号和一键回炉入口。
- 导出的 Markdown 必须包含同样信息，方便用户离线复盘。
- 不新增后端接口，不引入登录、付费或远程依赖，保持完全免费和本地可用。

## 非目标

- 不做图表库引入，避免增加包体和测试复杂度。
- 不跨全站历史数据做画像，本功能只回答“这一轮练习暴露了什么能力短板”。
- 不改变现有错因账本、验收和下一轮训练排序规则。

## 方案

新增 `PracticeSessionAbilityRadar` 类型和 `buildPracticeSessionAbilityRadar(queue, progress)` 工具函数。函数先复用当前队列的最新尝试，再按 `feedback.criteria` 聚合维度。每个维度输出平均分、尝试数、低于 70 分的题号、状态和行动建议。

面板中在“本轮错因验收”和“下一轮训练”之间加入一个紧凑区块。区块显示最弱维度、四个维度分数条和主按钮。按钮优先跳到低分题队列；如果没有低分题但有已答题，则回到本轮队列继续加压；如果没有样本，则跳到 `/practice`。

Markdown 在战报里新增 `## 本轮薄弱能力雷达`，包含状态、最弱维度、四个维度概览和主行动。

## 状态规则

- `empty`：本轮没有任何模拟面试记录。
- `risk`：存在维度平均分低于 60。
- `watch`：存在维度平均分低于 70。
- `stable`：全部维度平均分不低于 70。

## 测试策略

- 工具测试覆盖低分维度聚合、空队列导出和 Markdown 输出。
- 面板测试覆盖渲染最弱维度、四维分数和按钮导航。
- 保持现有完整前端测试、生产构建和 `git diff --check` 作为提交前门槛。
