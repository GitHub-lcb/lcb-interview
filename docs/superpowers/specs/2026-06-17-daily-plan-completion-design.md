# 今日闭环验收设计

## 背景

学习计划页已经具备配速补齐、今日作战简报和今日队列训练入口。用户现在能知道“今天做什么、为什么做、从哪里开始”，但完成训练后还缺少一个闭环判断：今天是否已经达标，剩下的风险是什么，是否还需要补复习债、薄弱题或模拟面试。

要让免费备考体验更接近私人教练，需要在本地学习进度上生成“今日闭环验收”，把今日计划的完成度、复习风险和表达样本合成一个明确结论。

## 目标

- 基于 `StudyProgress` 和当前时间生成今日计划验收报告。
- 统计今日计划题数、已掌握题数、薄弱题数、计划内复习债、今日模拟面试样本数。
- 给出验收状态：待生成、进行中、有风险、已达标、已加强。
- 给出一个主行动：生成计划、清复习债、修复薄弱题、继续训练、补模拟面试或查看报告。
- 完全免费、本地计算，不新增后端接口。

## 非目标

- 不修改题目状态流转规则。
- 不新增每日历史归档。
- 不把验收结果写入本地存储；报告随学习状态实时刷新。

## 设计方案

新增 `buildDailyPlanCompletion(progress, now)`：

1. 去重读取 `progress.dailyPlan`。
2. 逐题读取 `QuestionStudyState`：
   - `mastered` 计入已掌握。
   - `weak` 计入薄弱风险。
3. 使用 `buildScheduledReviewQueue(progress, now)` 找出计划内到期/逾期复习债。
4. 统计计划内今日模拟面试次数。判断“今日”时使用当前日期的本地日历日，兼容无时区字符串。
5. 计算完成率和状态：
   - `empty`：今日计划为空。
   - `risk`：有计划内复习债或薄弱题。
   - `active`：还有未掌握题。
   - `ready`：计划题都已掌握但今日没有面试样本。
   - `excellent`：计划题都已掌握且已有今日面试样本。
6. 按状态生成主行动：
   - 空计划：生成今日计划。
   - 复习债：清复习债。
   - 薄弱题：修复薄弱题。
   - 未完成：继续今日队列。
   - 缺面试样本：补一次模拟面试。
   - 已加强：查看冲刺报告。

页面新增 `DailyPlanCompletionPanel`：

- 放在今日作战简报之后、面试简报之前。
- 显示完成率、风险指标和主行动。
- 有计划时展示最多 4 个待处理事项。

## 行为规则

- 今日计划为空时不制造完成率，完成率为 0。
- 计划中重复题只计算一次。
- 到期/逾期复习债优先于薄弱题，因为时间窗口更紧。
- 今日模拟面试只统计 `createdAt` 与 `now` 在同一日历日的尝试。
- 如果一个计划题没有状态，按 `new` 处理，视为未完成。

## 测试策略

- 单元测试覆盖空计划、复习债风险、薄弱风险、未完成、已达标缺面试、已加强。
- 组件测试覆盖指标和主行动渲染。
- 提交前运行前端全量测试、前端构建、后端测试和 `git diff --check`。
