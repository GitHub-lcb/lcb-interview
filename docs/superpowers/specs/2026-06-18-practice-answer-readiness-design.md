# 模拟面试回答实时结构检查设计

## 背景

`/practice` 已有答题脚手架，可以帮助用户从空白输入框开始组织答案。但用户把模板带入后，仍可能只保留结论、删除场景，或者写了一段很长但没有边界风险的答案。等到提交评分后才发现问题，反馈链路仍然偏慢。

本阶段增加“回答结构实时检查”，在用户输入时即时判断草稿是否覆盖结论、机制、场景和边界四段。它不替代正式评分，只做提交前的本地守门，让用户在点击“提交评分”之前就知道还缺哪一段。

## 目标

- 在 `/practice` 回答输入框下方展示实时结构检查。
- 检查四个维度：结论、机制、场景、边界。
- 根据当前题目关键词辅助判断机制覆盖，避免只检测固定口号。
- 输出总分、等级、下一步动作和每个维度的命中状态。
- 空回答、短回答、结构完整回答都要有可执行提示。
- 纯函数输出不得出现 `undefined`。

## 非目标

- 不改提交评分算法，不影响远端 AI 或本地规则评分。
- 不要求用户必须满分才能提交，避免阻断自由练习。
- 不新增后端接口，不保存实时检查结果。
- 不做完整语义理解，只做轻量规则检查。

## 检查规则

| 维度 | 命中信号 |
|------|----------|
| 结论 | 包含“结论/首先/我认为/核心是”，或回答开头已覆盖题目关键词 |
| 机制 | 包含题目关键词、标签关键词，或“机制/原理/底层/链路/流程/因为”等解释词 |
| 场景 | 包含“项目/线上/生产/高并发/例如/比如/在...场景” |
| 边界 | 包含“风险/边界/注意/不能/缺点/误区/兜底/降级/替代” |

分数按四段平均，每段 25 分：

- `empty`：0 分，没有可分析回答。
- `draft`：1-49 分，草稿仍缺主要结构。
- `ready`：50-74 分，已有基础结构但不稳。
- `sharp`：75-100 分，可以进入提交评分。

## UI 设计

- 放在回答输入框和提交按钮之间。
- 标题为“回答结构实时检查”，配进度条和分数。
- 四个维度用紧凑条目展示，命中显示绿色勾，缺失显示橙色提示。
- 下一步动作只提示最优先缺口，例如“先补一个线上/项目场景”。
- 移动端保持单列，不挤压提交按钮。

## 测试策略

- 新增 `practiceAnswerReadiness.test.ts`：
  - 空回答输出 `empty`、0 分和第一句结论提示。
  - HashMap 结构完整回答输出 `sharp`，四段都命中。
  - 只有结论和机制的回答提示补场景，不出现 `undefined`。
- 新增 `PracticeAnswerReadinessPanel.test.tsx`：
  - 空回答渲染实时检查标题和“先写出第一句结论”。
  - 结构完整回答渲染 “100” 和“可以提交评分”。
