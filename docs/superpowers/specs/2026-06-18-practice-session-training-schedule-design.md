# 模拟面试战报下一轮训练日程设计

## 背景

战报已经能给出本轮通过门槛、过线证据包、下一轮训练契约和个性化题组，但用户还需要知道“接下来什么时候做什么”。如果只给题组和目标，用户仍要自己拆时间块，容易继续停在收藏、复盘或犹豫阶段。

## 目标

在战报中新增“下一轮训练日程”，把训练契约拆成可执行的时间安排：

- 空队列时提示先完成一次模拟面试，不显示伪日程。
- 修复模式下先安排低分修复、二次提交稿和通过验收。
- 推进模式下安排新题预热、限时作答和证据复盘。
- Markdown 导出同步包含日程，方便用户复制到笔记或计划工具。

## 方案

采用前端纯派生模型，不新增后端接口和本地存储。工具函数基于现有 `buildPracticeSessionTrainingContract`、`buildPracticeSessionPassGate` 和 `buildPracticeSessionNextTrainingQueue` 生成 3 个日程块，每个日程块包含时间段、标题、任务说明、验收标准和跳转入口。

日程状态分为：

- `empty`：没有本轮样本，无法生成日程。
- `repair`：本轮尚未过线，优先修复低分和弱项。
- `advance`：本轮已可推进，进入下一轮个性化训练。

## 界面

在训练契约之后、下一轮训练题组之前展示日程。布局保持紧凑，桌面端三列展示，移动端单列堆叠。每个卡片突出时间段和验收标准，底部主按钮进入日程第一步。

## 测试

- 工具测试覆盖 Markdown 中的“下一轮训练日程”和日程条目。
- 空队列 Markdown 必须提示等待生成训练日程。
- 组件测试覆盖日程面板、关键文案和主按钮跳转。
- 运行聚焦测试、完整前端测试和构建。
