# 真实面试场景包导出设计

## 背景

`/experiences` 页面把题目放回真实面试语境，按公司类型、岗位深挖和终面表达组织训练。它能帮助用户从“单题会不会”升级到“面试官连续追问时能不能讲稳”。

当前问题是场景组只能在页面内查看。用户在面试前通常需要把不同公司类型、追问主题和行动入口整理到外部笔记或陪练清单里。如果不能一键导出，场景训练会变成手动抄写，降低完全免费产品的执行效率。

本阶段增加“真实面试场景包”Markdown 导出，让用户把场景题单、追问主题和入口一次性复制出去。

## 目标

- 导出目标岗位、生成日期、场景总览和所有面试场景组。
- 每个场景组包含公司类型、摘要、追问主题和行动入口。
- 空场景配置也要导出可执行提示，避免空白文档。
- 页面只负责复制和下载降级，导出内容由纯函数生成。
- 增加纯函数和页面交互测试，防止导出出现 `undefined` 或复制按钮失效。

## 非目标

- 不新增后端接口，不根据公司类型动态生成题目。
- 不导出完整题目答案，只导出场景训练组织方式和入口。
- 不改变现有场景组数据结构。

## 内容结构

```markdown
# Java 后端 真实面试场景包

生成时间：2026-06-18

## 场景总览
- 场景组：4 组
- 覆盖公司类型：一二线互联网、成长型团队...
- 主训练入口：/practice

## 场景题单
1. 大厂 Java 后端面试组
   - 公司类型：一二线互联网
   - 摘要：按真实面试节奏组合基础、项目...
   - 追问主题：自我介绍后的项目深挖、JVM 与并发追问...
   - 行动入口：刷后端场景题（/search?q=后端场景）；开始一轮练习（/practice）
```

空状态导出：

```markdown
## 场景题单
1. 暂无面试场景组
   - 下一步：先确认目标岗位，再补充至少一组场景题单。
   - 入口：/practice
```

## UI 设计

- 在 `/experiences` 头部说明下增加“复制场景包”按钮。
- 使用 `CopyOutlined` 图标，和路线页的“复制路线包”位置保持一致。
- 成功复制提示“真实面试场景包已复制”。
- 剪贴板不可用时下载 `{岗位}-真实面试场景包.md`，并提示“剪贴板不可用，已下载 Markdown 场景包”。

## 测试策略

- 新增 `experiencePlaybook.test.ts`：
  - 有场景组时导出标题、日期、总览、题单、追问主题和行动入口。
  - 空场景组时导出可执行提示。
  - 内容不出现 `undefined`。
- 新增 `Experiences/index.test.tsx`：
  - 写入本地目标岗位。
  - mock `navigator.clipboard.writeText`。
  - 点击“复制场景包”后断言写入 Markdown 包含标题、场景题单和 `/practice`。
