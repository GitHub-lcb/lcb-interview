# 模拟面试战报首题复盘验收卡设计

## 背景

首题复盘模板已经告诉用户下一轮第一题或回修首题完成后要写哪些字段，但模板本身仍不能判断复盘是否可用。用户可能填写了评分、证据、阻断和下一题动作，却没有确认这些内容是否足以支撑继续加压。

## 目标

在“首题复盘模板”和“下一轮训练建议”之间增加“首题复盘验收卡”。验收卡把复盘模板的 4 个字段转成可核验标准：评分可比、证据可追溯、阻断可判定、下一题可执行。所有内容继续由本地规则派生，保持完全免费使用。

## 方案

首题复盘验收卡由 `buildPracticeSessionFirstQuestionReviewTemplate` 派生：

- 复盘模板为空时，显示等待状态。
- 复盘模板处于回修状态时，显示“回修复盘验收卡”，强调阻断项是否减少。
- 复盘模板处于 ready 状态时，显示“首题复盘验收卡”，强调下一轮首题是否留下新样本。
- 每个验收点包含目标、检查口径和未通过补救动作。
- 主行动沿用复盘模板入口，避免新增路由或后端状态。

## 用户价值

相比只给复盘模板，验收卡让用户明确知道复盘记录是否能继续使用。它把“我写了复盘”提升为“我的复盘能证明分数、证据、阻断和下一题动作都闭环”，减少无效复盘和盲目刷题。

## 验收

- Markdown 导出包含“首题复盘验收卡”，且位于“首题复盘模板”之后。
- 前端战报展示 4 个验收点和主按钮。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
