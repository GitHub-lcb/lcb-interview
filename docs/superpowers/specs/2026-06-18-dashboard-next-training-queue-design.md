# 工作台日报下一轮训练队列设计

## 背景

工作台日报已经能导出工作台概览、今日闭环验收、下一题、今日计划题单和弱点雷达。但用户把日报复制到外部复盘文档后，仍需要回到页面里判断下一轮到底练哪些题。刚新增的下一轮训练队列已经把评分影响、复习债、面试错因、薄弱题、学习中题和今日计划题融合成一个可执行队列，因此日报应该直接带出这部分结果。

## 目标

- 在 `buildStudyDashboardMarkdown` 导出的 Markdown 中增加 `## 下一轮训练队列`。
- 展示队列状态、摘要、主行动和入口。
- 展示前 5 道训练题，包含来源、原因、行动和入口。
- 空队列时给出可执行提示，而不是导出空章节。

## 非目标

- 不新增页面布局。
- 不新增本地存储字段。
- 不复制完整队列工具逻辑，全部复用 `buildNextTrainingQueue`。

## 数据流

`StudyDashboard` 调用 `buildStudyDashboardMarkdown` 时传入当前 `StudyProgress`、热门题和时间。日报工具在现有今日闭环之后调用 `buildNextTrainingQueue(progress, now, 5)`，把队列章节放在 `今日闭环验收` 和 `下一题` 之间，让用户先看到闭环结论，再直接看到下一轮训练入口。

## 边界

- 队列为空：输出“暂无下一轮训练题”，并指向 `/practice` 或今日计划生成动作。
- 队列非空：只展示前 5 项，避免日报过长。
- 已掌握高分题：展示其沉淀行动，不误导为继续刷题。
- 输出不得包含 `undefined`。
