# 冲刺报告岗位能力地图导出设计

## 背景

首页已经有“岗位能力地图”，可以把本地学习轨迹按备考路线聚合成岗位能力画像。但冲刺报告目前只导出健康度、今日任务、面试急救包、表达素材、追问防线和错题恢复闭环，缺少路线级能力视角。用户把报告复制到笔记或发给同伴复盘时，仍然看不到“哪个岗位能力最拖后腿、下一批题目该从哪个路线进入”。

本阶段要让冲刺报告具备离线可携带的岗位能力地图，继续强化完全免费、无需登录、无需付费报告的体验。

## 目标

- 在 `buildSprintReportMarkdown` 中新增“岗位能力地图”章节。
- 复用 `buildAbilityMap(routes, progress)`，不重复实现能力画像算法。
- 导出每个能力域的准备度、已记录题数、掌握题、薄弱题、学习中题和训练入口。
- 只导出前 5 个最值得处理的能力域，避免 Markdown 报告过长。
- 当没有本地轨迹时也输出空状态和路线入口，避免导出空白章节。
- 在“下一步行动”中增加“能力地图”行动线，指向当前最值得补强的路线训练队列。

## 非目标

- 不新增后端接口。
- 不拉取全量题库。
- 不修改首页能力地图 UI。
- 不新增付费、登录或云端报告能力。

## 方案

### 方案 A：只在健康度四维诊断里保留能力覆盖

改动最小，但能力覆盖只有一行摘要，不能展示具体路线、题目队列和能力短板。

### 方案 B：在冲刺报告新增独立能力地图章节

复用已有 `buildAbilityMap`，导出路线级指标和训练入口。报告可独立指导离线备考，也能解释今日任务为什么推荐某个能力域。

### 方案 C：新增专门的报告能力算法

可以定制导出格式，但会与首页能力地图产生重复逻辑，后续维护成本更高。

选择方案 B。它能用最小实现成本补齐报告视角，并保持页面与导出报告使用同一套能力语言。

## Markdown 结构

新增章节位于“面试追问防线”之后、“备考健康度”之前：

```md
## 岗位能力地图
- 总览：当前最需要补强 Java 后端冲刺路线，准备度 62 分。
- Java 后端冲刺路线｜Java 后端：准备度 62 分，薄弱 1，道学习中 1，道已掌握 1，道已记录 3 道；短板明显，优先进入路线训练，把薄弱题拉回学习中。；入口：/practice?queue=1,3
- AI 应用开发路线｜AI 应用工程师：准备度 0 分，薄弱 0，道学习中 0，道已掌握 0，道已记录 0 道；还没有本地题目轨迹，先打开路线题目建立能力画像。；入口：/routes
```

空状态：

```md
## 岗位能力地图
- 总览：还没有岗位能力轨迹，先从目标路线进入题目建立样本。
- Java 后端冲刺路线｜Java 后端：准备度 0 分，薄弱 0，道学习中 0，道已掌握 0，道已记录 0 道；还没有本地题目轨迹，先打开路线题目建立能力画像。；入口：/routes
```

下一步行动新增：

```md
- 能力地图：训练 Java 后端冲刺路线 - 先补 2 道未掌握题，把岗位能力短板拉回学习中。（/practice?queue=1,3）
```

空状态下一步行动：

```md
- 能力地图：打开备考路线 - 先建立岗位能力样本，系统才会生成路线级短板。（/routes）
```

## 实现要点

- `sprintReport.ts` 引入 `AbilityMapItem` 和 `buildAbilityMap`。
- 在 `buildSprintReportMarkdown` 中计算 `abilityMap = buildAbilityMap(routes, progress)`。
- 新增 `renderAbilityMapSection(abilityMap)`。
- 新增 `buildAbilityActionLine(abilityMap)`，选择第一个有 `nextQuestionIds` 的能力项；如果没有，则回退到第一个能力项或 `/routes`。
- `renderActionSection` 增加 `abilityMap` 参数，并插入能力地图行动线。
