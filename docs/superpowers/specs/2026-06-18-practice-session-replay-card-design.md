# 本轮 60 秒复述卡设计

## 背景

“本轮证据缺口”能告诉用户哪里会被追问，但用户下一步仍然要自己组织语言。真正超越普通题库的体验应该把诊断直接变成可开口的重答脚本：先说什么、补什么证据、最后怎么收边界。这样用户不需要离开页面就能完成一次 60 秒复述。

## 目标

- 在本轮战报新增“本轮 60 秒复述卡”。
- 基于证据缺口生成最多 3 张口播卡。
- 每张卡包含开场句、证据句、边界句和复述提示。
- Markdown 导出包含同样内容，方便离线背诵和复盘。
- 不调用外部 AI，不新增后端接口，继续保持完全免费。

## 非目标

- 不生成长篇标准答案，避免用户背模板而不练开口。
- 不替代原有追问防线；复述卡只负责把当前缺口转成 60 秒表达骨架。
- 不持久化复述完成状态，后续由新一轮评分自然反馈。

## 方案

新增 `PracticeSessionReplayCards` 类型和 `buildPracticeSessionReplayCards(queue, progress)` 构建器。

生成规则：

1. 优先读取 `buildPracticeSessionEvidenceGaps` 的前 3 个缺口。
2. 按低分维度生成不同口播骨架：
   - 知识覆盖：定义、机制、替代方案。
   - 表达结构：结论、背景、方案、结果、边界。
   - 场景细节：项目场景、指标、个人动作。
   - 风险边界：失败场景、降级策略、取舍依据。
3. 没有证据缺口但已有答题时，生成“稳定复述卡”，用于保持通过表现。
4. 没有答题样本时展示等待状态。

面板位置放在“本轮证据缺口”之后、“下一轮训练”之前：先展示缺口，再给可直接开口的修复脚本。

## 测试策略

- 工具测试覆盖低分场景导出复述卡、开场句、证据句和边界句。
- 空队列导出等待提示。
- 面板测试覆盖复述卡列表和主行动导航。
- 提交前运行 focused tests、完整前端测试、生产构建和 `git diff --check`。
