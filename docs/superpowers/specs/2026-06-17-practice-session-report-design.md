# 本轮模拟面试战报设计

## 背景

Practice 页已经支持队列练习、单题评分、追问加压和面试后闭环教练，但用户完成一轮题目后仍缺少一个“本轮整体表现”的判断。用户只能逐题看反馈，很难快速知道这一轮平均水平如何、哪些题拖了后腿、下一步应该继续答未完成题还是优先补弱。

## 目标

- 完全免费，基于前端已有 `StudyProgress` 和当前练习队列生成战报。
- 给出本轮已答数、平均分、通过数、最薄弱维度和未答数。
- 根据本轮结果自动生成一个主行动：开始练习、继续未答、补弱重练或进入学习页复盘。
- 不新增后端接口，不改变评分、进度和题目数据结构。
- 面板常驻 Practice 侧栏，让用户在答题时能持续看到本轮进展。

## 方案

新增 `buildPracticeSessionReport(queue, progress)` 纯函数，输入当前练习队列和学习进度，输出 `PracticeSessionReport`：

- `level`：`empty`、`in-progress`、`risk`、`passed`。
- `title` / `summary`：解释当前轮次状态。
- `answeredCount` / `totalCount`：本轮已答和总题数。
- `averageScore`：仅统计当前队列中已有最新面试尝试的题目。
- `passCount`：最新分数大于等于 70 的题目数。
- `weakQuestionIds`：本轮低于 70 分或已被用户标弱的题目。
- `metrics`：用于 UI 展示的四个关键指标。
- `primaryAction`：主行动，包含按钮文案、说明和跳转路径。

判断策略：

- 队列为空或没有任何已答题：展示 `empty`，主行动为开始本轮练习。
- 存在低于 70 分题目，或队列中有标弱题目：展示 `risk`，主行动跳转到弱题队列。
- 还有未答题：展示 `in-progress`，主行动跳转到未答队列。
- 全部答完且平均分大于等于 80、无弱题：展示 `passed`，主行动跳转到学习页复盘。
- 其他完成但未达到优秀状态：展示 `in-progress`，主行动引导继续复盘。

## 页面呈现

新增 `PracticeSessionReportPanel`，接入 Practice 页右侧栏，放在现有练习统计面板之后。面板展示：

- 标题和总结，直接说明本轮处于补弱、推进还是通过状态。
- 四个紧凑指标：已答题、平均分、通过数、最弱项。
- 一个主行动按钮，点击后调用 Practice 页传入的 `onNavigate(to)`。

该面板不承担评分、不修改进度、不写入存储，只负责把已有进度转成可读战报和下一步入口。

## 测试

- 工具函数测试覆盖空队列、部分已答、低分补弱、高分通过。
- 组件测试覆盖标题、指标、主按钮渲染和按钮点击回调。
- Practice 页只做轻量接入，不新增复杂页面级测试。
