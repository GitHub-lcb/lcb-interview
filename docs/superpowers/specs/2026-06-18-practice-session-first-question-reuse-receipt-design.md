# 模拟面试战报首题复用回执模板设计

## 背景

首题归档复用清单已经把归档证据转成下一轮开场前的动作，但清单仍然停留在“提醒用户要做什么”。用户真正进入下一轮训练后，需要把是否读过分数、是否带入证据、是否承认阻断、是否打开下一题记录下来，否则下一轮复盘仍缺少可追溯证据。

## 目标

在“首题归档复用清单”和“下一轮训练建议”之间增加“首题复用回执模板”。模板把复用清单的 4 个动作转成可填写字段：分数已读、证据已带、阻断已认、下一题已开。所有内容继续由本地规则生成，保持完全免费使用。

## 方案

首题复用回执模板由 `buildPracticeSessionFirstQuestionArchiveReuse` 派生：

- 复用清单为空时，显示等待回执状态。
- 复用清单处于回修状态时，显示“回修复用回执模板”，强调是否真的带着阻断修复证据回到首题。
- 复用清单处于 ready 状态时，显示“首题复用回执模板”，强调是否把归档证据带入下一轮首题。
- 每个回执字段包含填写提示、示例和验收规则。
- 主行动复用复用清单入口，避免新增路由、后端状态或付费能力。

## 用户价值

相比只展示复用清单，回执模板让用户把“我准备怎么复用”推进到“我已经如何复用”。这能为下一次战报提供更完整的本地闭环证据，减少免费训练里常见的看完建议却没有记录执行结果的问题。

## 验收

- Markdown 导出包含“首题复用回执模板”，且位于“首题归档复用清单”之后、“下一轮训练建议”之前。
- 前端战报展示 4 个回执字段：分数已读、证据已带、阻断已认、下一题已开。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
