# 模拟面试战报首题归档复用清单设计

## 背景

首题复盘归档包已经把评分、证据、阻断和下一题种子整理成下一轮输入，但归档内容仍停留在“保存下来”。用户进入下一轮前，还需要知道这些归档片段如何被复用到开场、作答、追问和复盘，否则归档价值容易在下一轮训练中断开。

## 目标

在“首题复盘归档包”和“下一轮训练建议”之间增加“首题归档复用清单”。清单把 4 个归档项转成 4 个下一轮开场前的核销动作：读分数、带证据、认阻断、开下一题。所有内容继续由本地规则派生，不依赖后端、付费 AI 或第三方服务。

## 方案

首题归档复用清单由 `buildPracticeSessionFirstQuestionReviewArchive` 派生：

- 归档包为空时，显示等待复用状态。
- 归档包处于回修状态时，显示“回修归档复用清单”，强调先复用阻断修复证据。
- 归档包处于 ready 状态时，显示“首题归档复用清单”，强调把首题样本带入下一轮。
- 每个清单项包含复用动作、开场提示、验收口径和失败回退。
- 主行动复用归档包入口，避免新增路由或持久化状态。

## 用户价值

相比只保存归档包，复用清单让用户下一轮开练前知道该读什么、带什么证据、承认什么风险、从哪道题开始。它把“复盘结果”推进到“下一轮可执行输入”，继续强化完全免费的个性化训练闭环。

## 验收

- Markdown 导出包含“首题归档复用清单”，且位于“首题复盘归档包”之后、“下一轮训练建议”之前。
- 前端战报展示 4 个复用项：读分数、带证据、认阻断、开下一题。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
