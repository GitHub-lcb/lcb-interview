# 模拟面试评分状态同步提示设计

## 背景

评分后学习状态已经能自动收敛，但用户在练习页只看到分数和状态徽标变化，缺少明确解释。低分题自动加入计划、高分题自动掌握这类行为如果不提示，用户可能以为系统没有记录或需要再手动点击。

## 目标

- 评分反馈区展示状态同步提示。
- 文案和状态规则保持一致：
  - `< 70`：已自动标记薄弱，并加入计划。
  - `70 - 79`：已同步为学习中，建议继续复盘。
  - `>= 80`：已同步为已掌握，可沉淀表达。
- 提示只在已有评分反馈时展示，不影响未评分页面。
- 不新增后端接口和持久化字段。

## 交互

用户提交评分后，在“面试官评分”分数卡下方看到一行状态提示。例如：

```text
已自动标记薄弱，并加入今日计划继续补强。
```

按钮“标记薄弱”“已掌握”仍保留，允许用户覆盖自动判断。

## 验收

- 工具函数能按分数返回状态和中文提示。
- 练习页反馈区使用同一工具函数展示提示。
- `npm run test` 与 `npm run build` 通过。
