# 模拟面试战报补弱模板导出设计

## 背景

战报补弱动作现在可以在练习页内一键带入重答模板，但 Markdown 战报导出仍只包含维度、原因、动作和入口。用户复制战报到本地笔记或离线复盘时，还需要回到页面生成模板。

## 目标

- Markdown 战报中的每条补弱动作都带上可直接复用的重答模板。
- 模板复用 `buildPracticeSessionRepairDraft`，避免导出文案与页面预填文案分叉。
- 无评分薄弱题也要导出兜底模板。
- 保持现有战报结构，不影响题目队列和下一步行动段落。

## 输出形态

每条补弱动作下新增“重答模板”小节，按缩进保留多行模板：

```markdown
1. Java 面试题 1
   - 维度：结构化
   - 原因：最近评分 56 分，结构化 38 分拖低本轮表现。
   - 动作：先按“结论 -> 原因 -> 场景 -> 边界”重组结构。
   - 入口：/practice?question=1
   - 重答模板：
     补弱题目：Java 面试题 1
     ...
```

## 验收

- Markdown 包含“重答模板”。
- 结构化低分题模板包含“结论 / 原因 / 场景 / 边界”。
- 空队列战报不出现 `undefined`。
- `npm run test` 与 `npm run build` 通过。
