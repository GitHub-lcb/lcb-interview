# 备考路线战术包导出设计

## 背景

`/routes` 页面已经按岗位展示免费备考路线，并能把本地题目轨迹映射成路线完成度、计划内数量、薄弱题和下一组训练题。这个页面解决了“我该按什么方向刷”的问题，但路线内容和个人进度目前只能在页面里查看。

用户在真正备考时通常需要把路线同步到外部笔记、日历、群内打卡或陪练清单里。如果路线不能导出，用户就要手动复制阶段、覆盖方向和下一组训练入口。为了让免费产品形成更强的个人化执行闭环，本阶段增加“备考路线战术包”Markdown 导出。

## 目标

- 导出目标岗位、生成日期、路线总览和每条路线的个人化进度。
- 每条路线包含岗位、周期、摘要、完成度、记忆/跟踪/计划/薄弱指标、推进阶段、覆盖方向、下一组训练入口和兜底入口。
- 没有本地题目轨迹时仍输出可执行下一步，提示先搜索或打开题目建立样本。
- 页面只负责复制和下载降级，导出内容由 `routeProgress` 纯函数生成。
- 增加纯函数和页面交互测试，防止导出内容出现 `undefined` 或复制入口失效。

## 非目标

- 不新增后端接口，不抓取全量路线题目。
- 不改变现有路线排序和完成度计算。
- 不导出题目答案，只导出路线执行信息和入口。

## 内容结构

```markdown
# Java 后端 备考路线战术包

生成时间：2026-06-18

## 路线总览
- 核心路线：4 条
- 今日优先：Java 后端冲刺路线
- 已建立轨迹：2 条

## 路线战术
1. Java 后端冲刺路线
   - 岗位：Java 后端
   - 周期：21 天
   - 摘要：从 Java 基础、并发、JVM...
   - 路线完成度：33%
   - 进度：已记忆 3 道，已跟踪 2 道，计划内 1 道，薄弱 1 道
   - 推进阶段：基础与集合、并发与 JVM...
   - 覆盖方向：Java 基础、Java 并发...
   - 下一组训练：/practice?queue=2,3
   - 兜底入口：/search?q=Java
```

空轨迹也必须导出：

```markdown
1. Java 后端冲刺路线
   - 下一组训练：先搜索并打开几道题建立本地轨迹。
   - 兜底入口：/search?q=Java
```

## UI 设计

- 在 `/routes` 头部文案下增加“复制路线包”按钮。
- 使用 `CopyOutlined` 图标，按钮靠近路线说明，而不是塞进单张路线卡。
- 成功复制提示“备考路线战术包已复制”。
- 剪贴板不可用时下载 `{岗位}-备考路线战术包.md`，并提示“剪贴板不可用，已下载 Markdown 路线包”。

## 测试策略

- `routeProgress.test.ts` 增加 Markdown 导出测试：
  - 有本地题目轨迹时导出标题、日期、总览、路线、阶段、覆盖方向、训练入口。
  - 空轨迹时导出“先搜索并打开几道题建立本地轨迹”和兜底入口。
  - 内容不出现 `undefined`。
- `PrepRoutes/index.test.tsx` 增加页面交互测试：
  - 写入本地路线题目进度。
  - mock `navigator.clipboard.writeText`。
  - 点击“复制路线包”后断言写入 Markdown 包含路线战术和训练入口。
