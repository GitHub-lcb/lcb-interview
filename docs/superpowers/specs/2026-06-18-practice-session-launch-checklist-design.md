# 模拟面试战报启动执行清单设计

## 背景

战报已经能生成下一轮启动包，告诉用户回修或推进时第一步要做什么。但启动包仍偏“动作提示”，用户执行后缺少可复制的证据模板，也缺少复盘问题来判断启动是否真的完成。为了让学习闭环更强，需要把启动动作转成可核销的执行清单。

## 目标

新增“启动执行清单”，把启动包中的动作转成可打卡证据：

- 空队列时提示先建立启动样本。
- 修复态生成回修执行清单，强调阻断项、证据和重新验收。
- 推进态生成下一轮执行清单，强调打开入口、复读目标、完成第一题评分样本。
- Markdown 导出同步包含执行清单，方便用户复制到复盘记录。
- 主按钮跳转到启动包对应入口。

## 非目标

- 不保存用户勾选状态。
- 不新增后端接口。
- 不做自动计时、提醒或推送。

## 方案

新增纯前端派生模型 `PracticeSessionLaunchChecklist`。它基于 `buildPracticeSessionLaunchPacket` 生成清单项，每项包含：

- 阶段名称。
- 完成口径。
- 证据模板。
- 复盘问题。
- 入口和优先级。

状态沿用启动包语义：`empty`、`repair`、`ready`。这样启动包负责“做什么”，执行清单负责“怎么证明做完”。

## 界面

在“下一轮启动包”之后、“下一轮训练”之前展示启动执行清单。卡片使用紧凑列表，修复态警示，推进态通过，空态中性。移动端单列展示。

## 测试

- Markdown 导出包含“启动执行清单”“证据模板”“复盘问题”“完成口径”。
- 空队列 Markdown 包含“等待生成启动执行清单”。
- 组件测试覆盖执行清单面板、关键清单项和主按钮跳转。
