# 模拟面试战报下一轮启动包设计

## 背景

战报已经能给出下一轮准入闸门，告诉用户应该回修还是推进。但裁决之后仍然缺一个“开局动作包”：用户点击前需要知道打开哪个入口、第一步做什么、用什么结果判断本轮是否已经启动成功。

## 目标

新增“下一轮启动包”，把准入裁决转成可直接执行的开局动作：

- 空队列时提示先建立启动样本。
- 修复态给出回修启动包，强调先回到本轮队列清理阻断项。
- 推进态给出下一轮启动包，强调先打开下一轮队列、复读目标、完成第一题开口样本。
- Markdown 导出同步包含启动包，方便用户复制后照着执行。
- 主按钮跳转到对应入口。

## 非目标

- 不新增真实任务持久化。
- 不替代下一轮训练队列排序。
- 不做浏览器自动打开或后台任务调度。

## 方案

新增纯前端派生模型 `PracticeSessionLaunchPacket`。它基于 `buildPracticeSessionAdvanceGate` 和 `buildPracticeSessionNextTrainingQueue` 生成 3 个启动动作：

- `empty`：等待启动样本。
- `repair`：回修启动包。
- `ready`：下一轮启动包。

每个动作包含指令、完成口径、入口和优先级。修复态从准入闸门的未通过条件里取前置动作；推进态从下一轮训练队列里取第一题作为开局题。

## 界面

在“下一轮准入闸门”之后、“下一轮训练”之前展示启动包。卡片保持紧凑列表，修复态警示，推进态通过，空态中性。移动端单列显示。

## 测试

- Markdown 导出包含“下一轮启动包”“回修启动包”“打开启动入口”“完成口径”。
- 空队列 Markdown 包含“等待建立启动样本”。
- 组件测试覆盖启动包面板、关键动作文案和主按钮跳转。
