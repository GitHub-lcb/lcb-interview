# 本轮战报今日闭环快照设计

## 背景

本轮模拟面试战报已经能显示队列画像、补弱动作和下一轮训练建议，但用户完成一轮练习后仍然需要跳回学习计划页，才能知道今天是否真正闭环：计划完成率是多少、是否还有复习债或薄弱题、评分对今日计划产生了什么影响。

为了让免费产品形成更完整的自学闭环，战报需要在同一个位置给出今日闭环快照。用户看完本轮表现后，立即知道今天还差哪一步。

## 目标

- 在本轮战报 Markdown 中增加“今日闭环快照”章节。
- 在本轮战报面板中展示今日闭环标题、完成率、风险摘要和主行动。
- 复用 `buildDailyPlanCompletion`，不复制今日计划验收算法。
- 使用本轮练习队列补齐临时上下文，让未写入今日计划的队列也能参与战报内验收。

## 非目标

- 不替代学习计划页完整的“今日闭环验收”面板。
- 不改变 `DailyPlanCompletion` 的评分、复习债和待办算法。
- 不新增后端接口或存储字段。

## 方案

在 `practiceSessionReport.ts` 中新增 `buildPracticeSessionDailyCompletion(queue, progress, now)`，复用已有的本轮队列上下文补齐逻辑，再调用 `buildDailyPlanCompletion`。这样当前队列的题目信息、状态和评分可以参与当次战报，但不会写回用户真实进度。

Markdown 报告在“队列画像”之后插入“今日闭环快照”，导出状态、完成率、计划内风险、评分影响和主行动入口。

`PracticeSessionReportPanel` 在“队列画像”之后、“下一轮训练”之前显示一个紧凑的闭环快照：完成率、风险数、今日模拟样本、主行动按钮。该块只解释当天状态，不展开完整待办列表，避免战报面板过长。

## 验收标准

- 战报 Markdown 包含“## 今日闭环快照”、完成率、主行动入口和评分影响。
- 战报面板显示“今日闭环”、完成率和主行动按钮。
- 当前队列题目即使不在原始 `dailyPlan` 中，也能参与闭环快照。
- 空队列仍显示可执行的“生成今日计划”状态，不出现 `undefined`。
- `npm run test -- practiceSessionReport PracticeSessionReportPanel`、`npm run test`、`npm run build` 通过。
