# 今日闭环验收导出设计

## 背景

学习计划页已经有“今日闭环验收”，能判断今日题单是否完成、是否还有复习债或薄弱题、是否缺少模拟面试样本。它把“今天做了什么”和“还差哪一步”聚合成一个明确验收面板。

当前问题是验收结果只能停留在页面内。用户结束一天训练时，往往需要把完成率、风险项和下一步行动同步到个人笔记、群内打卡或明天的计划里。如果只能截图或手动抄写，会削弱完全免费产品的复盘闭环。

本阶段为今日闭环验收增加 Markdown 导出，让用户可以一键复制或下载当天验收报告。

## 目标

- 导出今日闭环等级、摘要、完成率、指标、待办项和主行动。
- 空计划、风险中、推进中、待面试验收、已加强五类状态都能导出可执行下一步。
- 剪贴板不可用时下载 Markdown，避免浏览器权限导致导出失败。
- 增加纯函数和组件交互测试，防止导出内容出现 `undefined` 或与页面状态分叉。

## 非目标

- 不新增后端存储，不把验收报告上传到云端。
- 不导出完整题目答案，只导出本地学习状态派生的验收信息。
- 不改变现有验收等级计算规则。

## 内容结构

导出文件使用以下结构：

```markdown
# Java 后端 今日闭环验收

生成时间：2026-06-18

## 验收概览
- 状态：今日闭环还有风险
- 摘要：今日 3 道计划题，已掌握 1 道...
- 完成率：33%

## 指标
- 完成率：33%，按已掌握题计算
- 已掌握：1/3，仍需推进
- 风险项：2 个，含复习债
- 今日面试：0 次，建议补一次

## 待办验收
1. 1 道计划题已到期或逾期
   - 说明：复习窗口已经打开...
   - 入口：/practice?queue=...

## 主行动
- 动作：先清复习债
- 说明：计划内存在到期或逾期题...
- 入口：/practice?queue=...
```

空状态也必须导出明确下一步：

```markdown
## 待办验收
1. 今日计划还没生成
   - 说明：先生成或补齐今日计划...
   - 入口：/study
```

## UI 设计

- 在 `DailyPlanCompletionPanel` 的分数区下方增加“复制验收”按钮。
- 使用 `CopyOutlined` 图标，按钮尺寸保持小型，避免压缩完成率数字。
- 成功复制提示“今日闭环验收已复制”。
- 剪贴板失败时下载 `{岗位}-今日闭环验收.md`，并提示“剪贴板不可用，已下载 Markdown 验收”。

## 测试策略

- `dailyPlanCompletion.test.ts` 增加 Markdown 导出测试：
  - 风险状态导出标题、日期、概览、指标、待办、主行动和入口。
  - 空状态导出可执行提示，且不出现 `undefined`。
- `DailyPlanCompletionPanel.test.tsx` 增加复制交互测试：
  - mock `navigator.clipboard.writeText`。
  - 点击“复制验收”后断言写入 Markdown 包含标题和当前验收状态。
