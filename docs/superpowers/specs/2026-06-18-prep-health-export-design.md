# 备考健康雷达导出设计

## 背景

首页“备考健康雷达”已经把复习债、岗位能力覆盖、模拟面试表达稳定性和执行节奏压缩成一个健康分。它比单纯题库更接近私人教练：用户可以快速知道今天最该补哪一块，以及应该进入哪个行动入口。

当前问题是健康诊断只能在页面内查看。用户无法把健康分、风险维度、四项诊断和主行动保存到个人复盘文档。对于“完全免费、个性化备考、学习闭环”的目标，健康雷达应该支持一键导出，形成“诊断 -> 行动 -> 复盘”的可携带证据链。

本阶段为备考健康雷达增加 Markdown 导出。

## 目标

- 导出目标岗位、生成日期、健康分、等级、摘要和主风险维度。
- 导出四个健康维度：复习债、能力覆盖、表达稳定性、执行节奏。
- 导出每个维度的分数、状态、指标、说明和细节。
- 导出主行动的动作、说明和入口。
- 空状态仍导出可执行下一步，不出现 `undefined`。
- 剪贴板不可用时下载 Markdown，保持完全免费的离线复盘路径。

## 非目标

- 不修改 `buildPrepHealthReport` 的评分、等级和主行动选择算法。
- 不新增后端接口或云端存储。
- 不导出完整题目答案或模拟面试原文。

## 内容结构

```markdown
# Java 后端 备考健康雷达

生成时间：2026-06-18

## 健康概览
- 状态：备考风险偏高
- 健康分：42
- 摘要：当前最需要处理的是复习债...
- 主风险：复习债，20 分

## 维度诊断
1. 复习债
   - 状态：危险
   - 分数：20
   - 指标：2 逾期
   - 说明：有 2 道题已经逾期...
   - 细节：逾期 2 道...

## 主行动
- 动作：处理复习债
- 说明：先复习逾期和薄弱题...
- 入口：/study
```

空状态导出：

```markdown
## 主行动
- 动作：建立学习轨迹
- 入口：/study
```

## UI 设计

- 在 `PrepHealthRadarPanel` 主行动区域增加“复制雷达”按钮。
- 使用 `CopyOutlined` 图标，保持和其他导出按钮一致。
- 成功复制提示“备考健康雷达已复制”。
- 剪贴板失败时下载 `{岗位}-备考健康雷达.md`。
- 移动端按钮保持左对齐，不挤压主行动按钮。

## 测试策略

- `prepHealth.test.ts` 增加 Markdown 导出测试：
  - 有风险状态导出标题、日期、概览、维度诊断、主行动和入口。
  - 空进度导出建立轨迹动作，且不出现 `undefined`。
- 面板复制逻辑与其他导出面板一致，使用构建和全量测试覆盖类型与引用。
