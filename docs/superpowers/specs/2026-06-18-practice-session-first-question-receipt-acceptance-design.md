# 模拟面试战报首题回执验收卡设计

## 背景

首题回执模板已经能引导用户记录动作、证据、是否达标和下一步，但模板本身还缺少验收动作。用户可能填写了回执，却没有确认内容是否足够支撑进入下一轮或继续回修。

## 目标

在“首题回执模板”和“下一轮训练建议”之间增加“首题回执验收卡”，把首题回执拆成 4 个验收点：动作明确、证据可查、结论可判、下一步可执行。所有内容由本地规则派生，继续保持完全免费。

## 方案

首题回执验收卡由 `buildPracticeSessionFirstQuestionReceipt` 派生：

- 回执模板为空时显示等待态。
- 回修态要求回执能说明阻断项是否减少。
- 推进态要求回执能说明首题是否形成新的评分样本。
- 每个验收点包含检查目标、通过口径和未通过补救动作。
- 主行动沿用首题回执入口，让用户回到同一入口补齐缺口。

## 验收

- Markdown 导出包含“首题回执验收卡”，位置在“首题回执模板”之后。
- 前端面板展示 4 个验收点和主按钮。
- 空会话显示等待态且不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
