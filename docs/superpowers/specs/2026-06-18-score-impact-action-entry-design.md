# 评分影响行动入口设计

## 背景

今日闭环和冲刺报告已经解释了评分为什么改变计划，但用户读完后还需要自己找到题目入口。对低分题来说，最自然的下一步是立刻重答补强；对高分题来说，最自然的下一步是打开题目，把表达沉淀为素材。解释如果没有行动入口，闭环仍然不够短。

## 目标

- 每条评分影响都带行动文案和目标入口。
- 低分 `weak` 进入单题练习队列，行动文案为“重答补强”。
- 中等分 `learning` 进入单题练习队列，行动文案为“继续复盘”。
- 高分 `mastered` 进入题目详情，行动文案为“沉淀题目”。
- 今日闭环面板中的评分影响项可点击。
- 今日闭环 Markdown 和冲刺报告都输出行动入口。

## 非目标

- 不新增路由。
- 不改变评分状态阈值。
- 不新增后端字段。

## 交互

评分影响列表从纯信息块升级为按钮式条目：

```text
HashMap 为什么线程不安全    55 分
已自动标记薄弱，并留在今日计划继续补强。
重答补强  ->
```

点击后进入 `/practice?queue=<questionId>`。高分题进入 `/question/<questionId>`，便于回看标准答案和沉淀表达。

## 验收

- `DailyPlanCompletionImpact` 输出 `actionLabel` 和 `to`。
- 低分、学习中、高分三类状态的入口和文案正确。
- 面板能展示行动文案，并能从同一数据结构导航。
- 今日闭环 Markdown 和冲刺报告包含行动入口。
- `npm run test` 与 `npm run build` 通过。
