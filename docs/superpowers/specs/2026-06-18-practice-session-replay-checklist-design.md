# 本轮复述验收清单设计

## 背景

60 秒复述卡能让用户直接开口重答，但说完后仍需要判断“这次是否真的修好了”。普通题库通常只给答案，不给练后验收标准。战报应该把复述卡继续压缩成一份可自查清单，帮助用户在重新提交前完成自检。

## 目标

- 在本轮战报新增“本轮复述验收清单”。
- 基于复述卡生成 3-4 个必须达标的自查项。
- 每个自查项包含验收点、失败信号和达标口径。
- Markdown 导出和面板展示保持一致。
- 不新增后端接口，不依赖外部服务，继续保持本地免费。

## 非目标

- 不引入交互式勾选状态，先提供静态验收标准。
- 不替代自动评分；清单用于提交前自查，自动评分用于提交后验证。
- 不生成长篇答题模板。

## 方案

新增 `PracticeSessionReplayChecklist` 类型和 `buildPracticeSessionReplayChecklist(queue, progress)` 构建器。

生成规则：

1. 复用 `buildPracticeSessionReplayCards` 的主行动入口。
2. 有复述卡时生成四类验收项：
   - 结论：是否 10 秒内给出明确判断。
   - 证据：是否有项目场景、指标或个人动作。
   - 边界：是否说明失败场景、风险或兜底。
   - 时长：是否能在 60 秒内完整表达。
3. 无复述卡时展示等待状态，引导先完成模拟面试。

面板位置放在“本轮 60 秒复述卡”之后、“下一轮训练”之前：用户先照卡说，再按清单自检，最后进入下一轮训练。

## 测试策略

- 工具测试覆盖低分场景导出验收清单。
- 空队列导出等待提示。
- 面板测试覆盖清单和主行动导航。
- 提交前运行 focused tests、完整前端测试、生产构建和 `git diff --check`。
