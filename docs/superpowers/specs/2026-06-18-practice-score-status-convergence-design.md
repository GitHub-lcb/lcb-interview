# 模拟面试评分后状态自动收敛设计

## 背景

模拟面试评分已经能记录每次回答和分数，但题目的学习状态仍主要依赖用户手动点击“标记薄弱”或“已掌握”。这会让学习计划、复习队列、战报和能力地图滞后于真实评分结果，用户完成一次评分后还要维护一次状态。

## 目标

- 每次记录模拟面试评分时，同步更新题目学习状态。
- 低分题自动进入薄弱和计划，形成免费可用的补弱闭环。
- 通过但不稳定的题从薄弱回到学习中，避免永久卡在薄弱。
- 高分题自动进入已掌握，提升每日计划完成度和能力地图可信度。

## 状态规则

| 分数 | 状态 | 计划处理 | 说明 |
|------|------|----------|------|
| `< 70` | `weak` | 加入计划 | 低于通过线，需要继续补弱。 |
| `70 - 79` | `learning` | 保持原计划状态 | 已过线但不稳定，继续复习。 |
| `>= 80` | `mastered` | 保持原计划状态 | 表达稳定，可沉淀为掌握题。 |

每次评分都更新 `lastReviewedAt` 和 `reviewCount`，让复习调度能基于真实练习时间工作。

## 非目标

- 不自动删除每日计划中的题，避免用户看不到当天已完成内容。
- 不修改评分算法本身。
- 不新增后端接口，全部基于本地 `StudyProgress`。

## 验收

- `recordInterviewAttempt` 对低分、通过、高分分别更新状态。
- 低分题会自动 `addedToPlan: true` 且进入 `dailyPlan`。
- 只保留最近 5 次尝试的既有行为不变。
- `npm run test` 与 `npm run build` 通过。
