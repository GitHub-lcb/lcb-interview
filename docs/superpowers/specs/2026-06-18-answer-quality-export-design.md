# 单题答案质量导出设计

## 背景

题目详情页的“答案质量”面板已经能给出单题可复述分数、已覆盖模块、缺失模块、面试官追问和误区提醒。它让用户不只是看答案，而是知道这道题在真实面试里还缺哪些表达模块。

当前这些诊断只能留在页面里。用户如果想把单题短板带到外部笔记、错题本或面试前速查清单，需要手动复制分数、追问和缺失模块。为了继续强化“完全免费、个性化备考、学习闭环”的目标，本阶段为答案质量面板增加 Markdown 导出，让单题训练结果可以沉淀到离线复盘文档。

## 目标

- 导出题目标题、分类、难度、生成日期、质量分和质量等级。
- 导出快速回答，方便用户保存 30 秒口径。
- 导出已覆盖模块和可补强模块。
- 导出面试官追问清单。
- 导出误区提醒。
- 空字段仍输出可执行提示，不出现 `undefined`。
- 面板提供“复制质量”按钮，剪贴板不可用时降级下载 Markdown。

## 非目标

- 不改变 `calculateAnswerQuality` 的评分权重和等级规则。
- 不导出完整标准答案全文，避免把题目详情页变成大段答案复制器。
- 不新增后端接口、账号体系或云端同步。

## 内容结构

```markdown
# HashMap 为什么线程不安全？ 答案质量卡

生成时间：2026-06-18

## 质量概览
- 分类：Java 集合
- 难度：MEDIUM
- 质量分：72
- 等级：可用
- 快速回答：HashMap 线程不安全主要来自...

## 已覆盖模块
- 30 秒口径
- 标准回答

## 可补强模块
- 原理深化
- 项目落地

## 面试官追问
1. HashMap 的底层机制是什么？
2. 如果面试官结合项目追问 HashMap，你会怎么落地回答？

## 误区提醒
- 不要只背结论，要补充原因、适用边界和项目中的取舍。
```

## UI 设计

- 在 `AnswerQualityPanel` 顶部质量卡中增加“复制质量”按钮。
- 使用 `CopyOutlined` 图标，保持和其他导出按钮一致。
- 成功复制提示“答案质量卡已复制”。
- 剪贴板不可用时下载 `{题目标题}-答案质量卡.md`。
- 按钮采用小尺寸，避免压缩质量分和进度条。

## 测试策略

- `answerQuality.test.ts` 增加 Markdown 导出测试：
  - 完整题目导出标题、日期、概览、覆盖模块、补强模块、追问和误区。
  - 缺字段题目仍导出可补强模块，不出现 `undefined`。
- 新增 `AnswerQualityPanel.test.tsx`：
  - 点击“复制质量”会写入剪贴板。
  - 剪贴板内容包含答案质量卡、质量概览和面试官追问。
