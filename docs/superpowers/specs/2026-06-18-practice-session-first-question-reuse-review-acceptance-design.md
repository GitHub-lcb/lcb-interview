# 模拟面试战报首题复用复盘验收卡设计

## 背景

首题复用复盘模板已经能要求用户记录复用后的分数、证据、阻断和下一题反馈，但模板本身还缺少验收口径。用户可能填写了复盘内容，却没有判断这些内容是否足以支撑下一轮训练调整，导致复用链路又停在“记录完成”，没有变成可执行的质量门槛。

## 目标

在“首题复用复盘模板”和“下一轮训练建议”之间增加“首题复用复盘验收卡”。验收卡把 4 个复盘字段转成可检查项：分数验收、证据验收、阻断验收、回流验收。所有判断继续由本地规则派生，不依赖后端、账号、付费 AI 或第三方服务。

## 方案

首题复用复盘验收卡由 `buildPracticeSessionFirstQuestionReuseReviewTemplate` 派生：

- 复盘模板为空时，显示等待验收状态。
- 复盘模板处于 repair 状态时，显示“回修复用复盘待验收”，重点检查未放行项是否补齐。
- 复盘模板处于 ready 状态时，显示“首题复用复盘待验收”，重点检查复用结果是否能回流下一题。
- 每个验收项包含验收目标、通过信号、缺失风险和补救动作。
- 主行动复用复盘模板入口，确保 CTA 总是回到已存在的练习路径。

## 用户价值

相比只有复盘模板，验收卡会把“我记录了什么”进一步压成“这份记录能不能用”。它让免费模拟面试战报不只是追加内容，而是持续把每一次复用变成可判断、可补救、可回流的训练证据。

## 验收

- Markdown 导出包含“首题复用复盘验收卡”，且位于“首题复用复盘模板”之后、“下一轮训练建议”之前。
- 前端战报展示 4 个验收项：分数验收、证据验收、阻断验收、回流验收。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
