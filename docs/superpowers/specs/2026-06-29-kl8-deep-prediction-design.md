# 快乐8深度预测设计

## 背景

现有快乐8选5推荐只基于热号、冷号、遗漏、区间和奇偶生成摘要，再交给 AI 直接输出 5 组号码。用户希望预测更全方位，时间和 token 消耗不是主要约束，并参考 MiroFish 的多阶段预测流程。

MiroFish 的可借鉴点不是具体彩票算法，而是流程：先把种子数据结构化，再构造多维视角，最后由报告 Agent 汇总推演结果。本设计保留本项目现有 Spring Boot、MyBatis-Plus、React、Ant Design 和 AI 配置，不引入 OASIS、Zep 或新的重依赖。

## 目标

- 后端生成更厚的结构化历史分析，覆盖号码、组合、分布、趋势、遗漏和风险。
- AI 推荐从“直接猜号码”升级为“基于候选池和多维报告的选择与反证”。
- 同步开奖或生成新推荐时，自动结算历史推荐对下一期开奖的命中情况，并用最近命中反馈校准下一次候选权重。
- 每次仍返回 5 组号码，每组 5 个号码，继续保留娱乐参考风险提示。
- 历史推荐需要保存深度分析结果，并兼容旧记录。
- 数据库迁移必须幂等，不覆盖既有生产数据。

## 非目标

- 不承诺提高中奖概率，不提供投注金额或投注策略。
- 不复制 MiroFish 源码，不引入其 AGPL 代码依赖。
- 不把快乐8模块做成完整异步仿真平台，本次保持同步生成体验。

## 后端设计

新增深度分析结构，核心包括：

- 号码档案：每个号码的频率、近期趋势、当前遗漏、平均遗漏、最大遗漏、波动、最近命中、区间、奇偶、尾数、综合分、标签。
- 多窗口统计：近 30、60、100、200 和实际请求期数窗口，自动按已有开奖数据裁剪。
- 组合关系：号码共现对、共现 lift、相邻号、连号、重号、上一期重复等。
- 候选池：按综合分筛出强候选，并标记热号、冷号、遗漏回补、趋势上行、均衡补位等角色。
- 命中反馈：对已生成推荐等待下一期开奖后回填命中号码、单组命中和累计命中；下一次推荐按热号、冷号、高遗漏、趋势、均衡五类信号的历史命中率生成 0.75-1.25 的权重倍率。该校准只和 25% 单号随机基线比较，用于抑制近期表现差的信号、放大近期表现好的信号，不作为中奖承诺。

AI 调用继续使用 `AiRuntimeConfigService` 的现有 OpenAI 兼容配置。提示词改为深度 JSON 输入，要求模型输出：

- `groups`：5 组推荐号码和更充分理由。
- `analysis`：中文分析章节，包括整体判断、特征信号、组合逻辑和风险提示。
- `confidenceLabel`：仅能是 `低`、`中低`、`中`，避免暗示确定性。
- `riskWarnings`：明确说明彩票随机性和统计局限。

若 AI 不可用或输出不合规，规则兜底使用同一候选池生成 5 组，来源标记为 `RULE_BASED`，并保留深度分析摘要。

## 数据库设计

在 `lottery_kl8_recommendation` 上增加可空字段：

- `analysis_json`：AI 或规则生成的深度分析 JSON。
- `candidate_pool_json`：本次候选池和关键号码档案 JSON。
- `calibration_snapshot_json`：本次推荐使用的策略校准快照 JSON。
- `strategy_version`：策略版本，例如 `KL8_DEEP_FEEDBACK_V2`。
- `evaluated_issue_no`、`evaluated_draw_date`、`hit_summary_json`、`total_hit_count`、`max_hit_count`、`evaluated_at`：推荐命中结算字段。

初始化脚本同步新增字段。部署迁移脚本使用幂等 `ALTER TABLE`，只加字段，不删除、不重建、不清空。

## 前端设计

快乐8面板保留同步、历史、推荐按钮，推荐结果区域增加：

- 推荐组合：显示 5 组号码、来源、置信标签和组内理由。
- 深度分析：展示整体分析、关键特征、组合策略和风险提示。
- 候选池：展示高分候选号码及标签。
- 命中反馈：展示历史推荐在下一期开奖中的命中结果、命中号码高亮和当时使用的策略校准倍率。
- 历史开奖：保留现有近期列表。

旧历史记录没有 `analysisJson` 和 `candidatePoolJson` 时，继续显示 `featureSummary`。

## 测试与验证

- 后端单测先覆盖深度特征计算和候选池构造。
- 后端单测覆盖命中结算、历史反馈校准，以及校准倍率对号码综合分的影响。
- 推荐策略单测覆盖 AI 深度 JSON 校验、旧 JSON 兼容和规则兜底。
- 构建验证运行 `mvn test` 和 `npm run build`。
