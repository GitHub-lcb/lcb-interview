# 模拟面试战报首题验收尺设计

## 背景

首题预演卡解决“第一题怎么开口”的问题，但用户开口后仍需要一把很短的尺子判断是否真的达标。否则用户可能只点进题目，却没有留下评分样本、证据和下一步决策。

## 目标

在“首题预演卡”和“下一轮训练建议”之间增加“首题验收尺”，把首题预演拆成 4 个必须核验的硬标准：开场命中、通过信号、证据留存、下一步明确。用户无需付费和外部服务，即可用本地规则完成首题前后的自查。

## 方案

首题验收尺由 `buildPracticeSessionFirstQuestionRehearsal` 派生：

- 预演为空时显示等待态，提示先生成首题预演。
- 预演为回修态时，验收尺聚焦第一项阻断是否真实修掉。
- 预演为推进态时，验收尺聚焦第一题是否形成可评分样本。
- 每个验收点包含目标、检查口径、证据要求和优先级。
- 主行动沿用首题预演入口，避免出现多个互相竞争的跳转。

## 验收

- Markdown 战报在“首题预演卡”之后、“下一轮训练建议”之前导出首题验收尺。
- 前端面板同样按该顺序展示 4 个验收点。
- 空会话显示等待态且不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
