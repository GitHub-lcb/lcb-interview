# 本轮面试官决策卡设计

## 背景

当前练习战报已经能展示错因、验收和能力雷达，但这些仍偏“学习复盘”。用户真正关心的是：如果这轮回答发生在真实面试里，面试官会给什么结论，阻断项是什么，下一步怎么把结论扭转。

## 目标

- 在本轮战报中新增“本轮面试官决策卡”。
- 用当前队列最新模拟面试记录生成面试官视角结论：等待样本、暂不建议通过、待观察、通过信号、强通过信号。
- 明确展示决策证据、阻断项和主行动。
- Markdown 导出包含同样信息，方便用户离线复盘或发给教练。
- 不新增后端接口，不依赖外部服务，继续保持完全免费和本地可用。

## 非目标

- 不模拟真实公司招聘流程，也不承诺录用概率。
- 不引入 AI 远程评审；所有判断只来自本地评分、队列进度和能力雷达。
- 不替代错因账本、能力雷达和下一轮训练，只作为“面试官视角总判定”。

## 方案

新增 `PracticeSessionInterviewerDecision` 类型和 `buildPracticeSessionInterviewerDecision(queue, progress)` 函数。函数复用 `buildPracticeSessionReport` 与 `buildPracticeSessionAbilityRadar` 的结果，避免重复计算评分和薄弱维度。

决策卡输出：

- `status`：`empty`、`reject-risk`、`hold`、`pass`、`strong-pass`。
- `title`：面试官视角结论。
- `verdict`：短句判定，例如“暂不建议通过”。
- `summary`：一句解释。
- `evidence`：已答题、平均分、通过题、最弱维度。
- `blockers`：未答题、低分题、能力雷达阻断项。
- `primaryAction`：优先跳转到最能改变结论的训练入口。

面板中放在“本轮薄弱能力雷达”之后、“下一轮训练”之前。这样用户先看到学习侧原因，再看到面试官侧结论，最后进入下一轮训练。

## 决策规则

- `empty`：无队列或没有本轮作答。
- `reject-risk`：平均分低于 70，或能力雷达处于 `risk`，或低分/薄弱题存在。
- `hold`：有未答题，或平均分达到 70 但未达到 80。
- `pass`：本轮已答完、无低分题、平均分达到 80。
- `strong-pass`：本轮已答完、无低分题、平均分达到 88，且能力雷达稳定。

## 测试策略

- 工具测试覆盖低分决策、空样本决策和 Markdown 输出。
- 面板测试覆盖结论、阻断项和主按钮导航。
- 提交前运行 focused tests、完整前端测试、生产构建和 `git diff --check`。
