# 模拟面试战报首题复用复盘模板设计

## 背景

首题复用放行门禁已经能判断复用回执是否可以进入下一轮，但放行之后仍缺少复盘字段。用户可能带着归档证据进入下一题，却没有记录复用是否提升分数、命中证据、回收阻断或反向修正下一题安排，导致复用链路停在“放行”而没有形成新的训练样本。

## 目标

在“首题复用放行门禁”和“下一轮训练建议”之间增加“首题复用复盘模板”。模板把复用放行裁决转成 4 个复盘字段：分数回看、证据命中、阻断回收、下一题回流。所有内容继续由本地规则派生，不依赖后端、账号、付费 AI 或第三方服务。

## 方案

首题复用复盘模板由 `buildPracticeSessionFirstQuestionReuseReleaseGate` 派生：

- 复用放行门禁为空时，显示等待复盘状态。
- 复用放行门禁处于 blocked 状态时，显示“回修复用复盘模板”，强调先复盘未放行项。
- 复用放行门禁处于 ready 状态时，显示“首题复用复盘模板”，强调复用证据进入下一题后的反馈。
- 每个字段包含填写提示、示例和验收规则。
- 主行动复用复用放行门禁入口，避免新增路由、存储或付费能力。

## 用户价值

相比只告诉用户能不能放行，复盘模板要求用户记录“复用之后发生了什么”。它让免费模拟面试链路形成新的反馈样本，后续训练不只是继续刷题，而是能看到复用证据对分数、追问和下一题安排的实际影响。

## 验收

- Markdown 导出包含“首题复用复盘模板”，且位于“首题复用放行门禁”之后、“下一轮训练建议”之前。
- 前端战报展示 4 个复盘字段：分数回看、证据命中、阻断回收、下一题回流。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
