# 持续超越面试鸭作战系统设计

## 背景

目标不是复制 mianshiya.com，而是把 LCB Interview 做成更高效的免费面试备考系统：用户进站后不用猜下一步，不被会员墙打断，能在一次会话内完成“选方向 -> 得到题单 -> 开口训练 -> 评分反馈 -> 补弱复盘 -> 导出材料”的闭环。

2026-06-20 重新抽样竞品公开页面：

- 首页包含题库、刷题路线、AI 面试、真实面经、会员和 VIP 标识。
- About 页面强调 1w+ 高频题、原创题解、持续更新、沉浸式学习、小程序、50 万+用户等能力。
- 热门分类页继续展示题库导航、真实面经和会员入口。

本站当前已有 6386 道题、46 个分类、结构化答案、学习计划、复习队列、模拟面试评分、本地进度、路线页、面经场景页、导出型复盘材料和管理后台质量治理。下一阶段的核心矛盾不是“有没有功能”，而是“新用户是否能在 3 分钟内完成一次有效训练，并愿意明天继续回来”。

## 北极星

**3 分钟有效训练率**：新用户从打开首页到完成第一道题的训练动作不超过 3 分钟。

有效训练定义：

1. 明确目标岗位或路线。
2. 进入一组具体题目，而不是停在列表浏览。
3. 至少完成一个动作：加入计划、标记薄弱、提交模拟回答、复制复盘材料。
4. 系统给出下一步，不让用户自己猜。

## 竞争指标

| 维度 | 竞品基线 | LCB 目标 | 验收方式 |
| --- | --- | --- | --- |
| 题量覆盖 | 公开宣称 1w+ 高频题 | 先达到 10000+，再按岗位图谱补齐缺口 | 后台质量仪表盘统计 PUBLISHED 题量、分类覆盖、空答案风险 |
| 题解质量 | 原创题解、图文并茂、可应对追问 | 每题至少包含 30 秒口径、原理、对比、场景、风险、项目表达 | AdminQualityService 风险项归零或低于阈值 |
| 免费体验 | 存在会员/VIP 信号 | 核心题库、答案、路线、评分、复盘全部免费 | 前端文案和功能入口不出现付费解锁 |
| 训练闭环 | 刷题路线 + AI 面试入口 | 每天自动给出下一题、追问、补弱、战报和导出材料 | 首页、学习页、练习页形成互相可达闭环 |
| 新手路径 | 需要理解多个导航入口 | 首屏直接给出 3 个可执行入口 | 新用户空状态下首屏出现“开始训练/选择路线/生成计划” |
| 移动端 | 强调小程序碎片化学习 | Web/PWA 首屏和练习页移动端无横向溢出，关键操作一屏可达 | 360px、390px、768px 截图 QA |
| 性能 | 大型内容平台 | 首屏 JS/CSS 分包可控，核心页面 2 秒内可交互 | Vite build 体积、Lighthouse/浏览器性能抽样 |
| 可持续运营 | 长期更新 | 内容生产、审核、发布、风险修复闭环 | 管理后台质量队列 + 发布门禁 |

## 自动审批原则

本轮用户明确授权“自己审批后持续进行”。因此后续小步迭代默认由 Codex 自审批，但以下边界必须停下向用户确认：

- 引入账号、手机号、邮箱、云同步、埋点或任何个人数据上传。
- 引入付费、会员、广告、第三方登录或商业化文案。
- 大规模删除内容、重写数据库初始化脚本或迁移生产数据。
- 调整 AI 供应商、密钥、计费策略或外部 API 调用成本。
- 需要长期运行的外部服务、定时任务或会产生费用的基础设施。

可自动批准的改动必须同时满足：

1. 直接提升竞争指标中的至少一项。
2. 保持核心学习能力免费开放。
3. 变更范围可回滚，且不破坏当前未提交工作。
4. 有聚焦测试或构建验证。
5. 前端体验符合“专业备考工具”风格：信息密度高但清晰，不做营销式空页面。

## 持续循环

每一轮都按同一个节奏推进：

1. **基线检查**：读取当前工作区、构建状态、最近文档和竞品公开信号。
2. **选择一个瓶颈**：只选一个最影响北极星的用户摩擦，不同时铺开多个大系统。
3. **写短规格**：说明问题、用户、成功指标、非目标、风险和验收。
4. **自审批**：按上面的自动审批原则给出通过/驳回结论。
5. **TDD 实施**：先写失败测试，再写最小实现。
6. **体验 QA**：至少覆盖桌面和移动布局；必要时启动 dev server 检查。
7. **验证**：运行聚焦测试、前端构建；触及后端时运行 Maven 相关测试。
8. **记录结果**：更新计划、遗留风险和下一轮候选项。

## 路线图

### P0：保护当前工作

- 不覆盖当前未提交的后台质量治理改动。
- 所有新增工作优先落在新文档或新前端模块，避免和当前后台改动冲突。
- 全量测试超时不直接判失败，改用聚焦测试 + 构建确认基线。

### P1：3 分钟首练通道

目标：空状态新用户打开首页时，立刻知道该点哪里。

交付：

- 首页首屏增加“首练启动台”，聚合目标岗位、可用时间、首个动作。
- 有数据用户显示“继续今日队列”，空数据用户显示“选择路线/生成计划/开始模拟”。
- 行动按钮必须落到已有 `/routes`、`/study`、`/practice`，不新增后端依赖。

验收：

- 新用户空 localStorage 时，首屏能在不滚动或少滚动下看到开始训练入口。
- 已有进度用户能看到下一步训练而不是泛泛的功能列表。
- 移动端按钮不换行溢出。

### P2：内容质量超过题量焦虑

目标：把 6386 道题中低质量、空答案、缺模块题持续修到可发布水平，再扩到 10000+。

交付：

- 管理后台质量看板继续完善分类风险、发布门禁、AI 生成失败恢复。
- 每个分类有题解完整度、风险题数、待发布队列。
- 初始化数据和 AI 生成产物能被重复审计。

验收：

- 空答案、短答案、缺关键模块的 PUBLISHED 题低于阈值。
- 低质量 DRAFT 不能误发布。

### P3：真实面试训练系统

目标：从单题练习升级为真实面试节奏。

交付：

- 多题面试回合、首题破冰、追问脚本、回合战报、补弱重答。
- 项目经历深挖模板、HR 表达、系统设计白板式回答结构。
- 可导出面试前 24 小时简报和 30 分钟急救包。

验收：

- 用户完成一轮后得到明确裁决：通过、可继续、必须补弱。
- 战报能告诉用户下一轮从哪题开始、为什么。

### P4：移动端与离线携带

目标：不依赖小程序，也能满足碎片化学习。

交付：

- PWA 安装、离线缓存关键页面、移动端练习优化。
- Markdown 导出链路统一：路线包、日报、复习队列、战报、急救包。

验收：

- 390px 宽度下首页、路线、学习、练习、详情无关键遮挡。
- 断网时仍能查看已缓存材料和本地进度。

### P5：面经与社区护城河

目标：在质量治理成熟后再做 UGC，避免先引入审核负担。

交付：

- 真实面经投稿、匿名化、审核、标签化。
- 公司/岗位/题目三向关联。
- 反垃圾、敏感信息清理、版权风险处理。

验收：

- 未审核内容不公开。
- 面经能反向生成训练队列，而不是只做阅读内容。

## 第一轮自审批

决议：批准 P1“3 分钟首练通道”作为下一轮执行范围。

理由：

- 它直接服务北极星指标。
- 复用现有 `/routes`、`/study`、`/practice`，不触碰后端 schema。
- 能在当前未提交后台质量改动之外独立开发。
- 比继续堆战报模块更能改善新用户第一印象。

非目标：

- 不做账号系统。
- 不做竞品命名攻击或公开贬损。
- 不做新的 AI 调用。
- 不重构现有大量学习/练习模块。
