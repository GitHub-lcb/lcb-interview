# 今日计划评分影响说明设计

## 背景

模拟面试评分已经会自动同步题目状态：低分回到薄弱并加入今日计划，中等分保持学习中，高分标记已掌握。当前“今日闭环验收”只展示完成率、风险项和下一步动作，没有解释评分为什么改变了计划状态。用户在复制验收报告或查看学习页时，仍需要自己推断是哪一次评分导致今日计划变动。

## 目标

- 今日闭环验收展示当天计划内模拟面试评分对计划状态的影响。
- Markdown 导出包含同一份解释，便于复盘和外部记录。
- 只统计今日计划内、当天产生的模拟面试评分，避免历史评分和计划外题目污染今日验收。
- 同一题当天多次评分时，只展示最新一次，反映当前计划状态。
- 复用既有分数到状态的规则，避免和评分页文案分叉。

## 非目标

- 不新增后端接口。
- 不修改评分算法。
- 不引入历史趋势图或跨日统计。

## 方案

在 `DailyPlanCompletion` 中新增 `statusImpacts` 字段，每一项包含题目 ID、题目标题、分数、同步后的状态、评分时间和中文说明。`buildDailyPlanCompletion` 负责从 `progress.interviewAttempts` 中筛选今日计划内评分，并按最新评分时间倒序输出。

说明文案按状态聚焦“为什么计划变了”：

- `weak`：低分题已回到薄弱，并留在今日计划继续补强。
- `learning`：题目已同步为学习中，还需要继续复盘到 80 分以上。
- `mastered`：题目已同步为已掌握，计入今日完成。

页面面板在指标和主行动之间展示“评分影响”列表。Markdown 在指标之后追加同名章节；没有今日评分时展示一条空状态说明，提醒评分后会自动解释计划变化。

## 验收

- 每日闭环工具能输出今日计划内评分影响，并排除计划外或非当天评分。
- 同一题当天多次评分只保留最新一次。
- Markdown 包含“评分影响”章节和具体题目分数说明。
- 面板展示评分影响，复制验收时带出同一章节。
- `npm run test` 与 `npm run build` 通过。
