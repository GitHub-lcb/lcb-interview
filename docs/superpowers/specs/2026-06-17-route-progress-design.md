# LCB Interview 路线进度增强设计

## 背景

路线页已经解决了“用户不知道从哪里开始”的问题，但目前仍然偏静态介绍。要继续朝“远超普通题库”的方向推进，路线页需要知道用户已经学过什么、计划了什么、哪里还没有启动。

本期把路线页升级为个人路线面板：基于本地 `StudyProgress` 和题目快照，计算每条路线的覆盖题数、已跟踪题数、计划内题数、薄弱题数、掌握题数和完成度。

## 目标

- 每条备考路线显示个人进度，而不只是阶段说明。
- 用户能一眼看到哪条路线最需要推进。
- 当路线已有记忆题目时，可以一键把该路线候选题加入今日计划。
- 当路线没有记忆题目时，引导用户先搜索该路线方向。

## 技术设计

### 纯函数

新增 `frontend/src/utils/routeProgress.ts`：

- `buildRouteProgress(route, progress)`：计算单条路线进度。
- `buildRouteProgressList(routes, progress)`：批量计算并按“薄弱多、计划少、进度低”排序。

匹配逻辑：

- 题目快照的 `categoryName` 命中路线 `categories` 时，归属该路线。
- 题目标签命中路线 `categories` 时，也归属该路线。
- 只使用本地已记忆题目，不主动发请求，避免路线页一次加载造成后端压力。

### 页面集成

修改 `PrepRoutes`：

- 使用 `useStudyProgress()` 读取本地进度。
- 每张路线卡显示：完成度、已跟踪、今日计划、薄弱题。
- 有路线候选题时显示“加入今日计划”按钮。
- 进入训练时带上该路线的题目队列：`/practice?queue=...`。

### 注释要求

路线归属和排序规则必须有中文注释，解释为什么不一次性请求全量路线题目。

## 验收标准

- 新增 route progress 单测覆盖：
  - 分类命中路线；
  - 标签命中路线；
  - 完成度计算；
  - 路线排序优先薄弱路线；
  - 无题目路线安全返回空进度。
- 路线页展示个人进度数据。
- 现有 `npm run test`、`npm run build`、`mvn test` 通过。
- 提交信息使用中文。
