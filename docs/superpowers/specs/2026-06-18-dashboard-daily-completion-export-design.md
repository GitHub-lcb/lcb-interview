# 工作台日报闭环验收导出设计

## 背景

首页工作台日报目前只导出概览、下一题、今日计划题单和弱点雷达。今日闭环验收、模拟评分影响和行动入口已经在学习计划页与冲刺报告中具备，但从首页点击“复制日报”时仍会丢失这部分信息，导致用户拿到的日报不够闭环。

## 目标

- 工作台日报增加“今日闭环验收”章节。
- 复用 `buildDailyPlanCompletion`，不重复计算完成率、复习债、薄弱题和评分影响。
- 有评分影响时输出题目、分数、状态变化、行动和入口。
- 无评分影响时输出空状态说明。
- 保持首页页面布局不变，只增强复制出的 Markdown。

## 输出结构

```markdown
## 今日闭环验收
- 状态：今日闭环还有风险
- 完成率：0%
- 主行动：修复薄弱题，/practice?queue=1
- 评分影响：HashMap 为什么线程不安全？，55 分，已自动标记薄弱，并留在今日计划继续补强。行动：重答补强，入口：/practice?queue=1
```

## 验收

- `buildStudyDashboardMarkdown` 输出“今日闭环验收”章节。
- 章节包含完成率、主行动和评分影响行动入口。
- 首页组件复制日报时带出同一章节。
- 空进度也有可读空状态，不出现 `undefined`。
- `npm run test` 与 `npm run build` 通过。
