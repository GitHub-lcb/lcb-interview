# 面试前急救包报告导出设计

## 背景

学习计划页已经新增“面试前急救包”，能把复习债、低分错因、薄弱开口和今日闭环压缩成 30 分钟内的临场行动队列。但复制出去的冲刺报告还没有包含这部分内容。用户面试前常会把报告粘到个人文档或手机备忘录，如果急救包只在页面里展示，就会削弱离线复盘和临场携带能力。

为了让免费产品继续向私人教练靠拢，冲刺报告需要同步导出急救包，让用户在没有打开网站的情况下也能看到最后一轮行动顺序。

## 目标

- 在 `buildSprintReportMarkdown` 中新增“面试前急救包”章节。
- 复用 `buildInterviewEmergencyKit(progress, now)`，不重复实现临场排序和预算规则。
- 报告导出急救包等级、摘要、总耗时、复习债数量、错因数量和最多 5 条行动。
- 每条行动保留标题、耗时、说明、原因和入口。
- 保持完全前端、本地计算，不新增接口。

## 非目标

- 不改变急救包页面组件。
- 不改变冲刺报告复制按钮。
- 不把急救包动作重新写入本地状态；报告始终实时计算。

## 设计方案

`buildSprintReportMarkdown(routes, progress, now)` 新增：

```ts
const emergencyKit = buildInterviewEmergencyKit(progress, now)
```

章节顺序调整为：

1. 基础信息
2. 一页作战摘要
3. 面试前急救包
4. 备考健康度
5. 四维诊断
6. 今日计划闭环
7. 今日作战简报
8. 面试简报与错题恢复

新增 `renderEmergencyKitSection(kit)`：

- 输出 `## 面试前急救包`
- 输出状态、摘要、预计耗时、复习债、错因信号
- 输出最多 5 条行动：`- 12 分钟｜先清复习债：...（入口）`

## 行为规则

- 空进度也导出急救包，提示先建立模拟面试样本。
- 行动为空时输出“暂无急救动作”，但正常生成器会提供兜底热身动作。
- 入口为空时降级为 `/practice`。
- 不能出现 `undefined`。

## 测试策略

- 扩展 `sprintReport.test.ts`：
  - 非空报告断言包含“面试前急救包”“面试前先压最高风险”“预计耗时：24 分钟”“先清复习债”。
  - 空报告断言包含“先建立临场样本”和 `/practice`。
- RED 阶段运行 `npm run test -- sprintReport`，确认测试因缺少急救包章节失败。
- GREEN 阶段接入急救包生成器和渲染函数。
- 提交前运行前端全量测试、前端构建、后端测试、`git diff --check` 和 HTTP 200 探测。
