# 今日闭环报告导出设计

## 背景

冲刺报告已经能导出备考健康度、四维诊断、面试表达优势、风险和热身题。近期新增了配速计划、今日作战简报、今日队列训练入口和今日闭环验收后，用户在页面内已经能看到“今天做什么、做完没有、下一步补哪里”。但复制报告仍停留在旧结构，离线分享给自己、同伴或面试教练时缺少今日闭环信息。

为了让免费产品具备更强的复盘迁移能力，报告导出需要把今日计划状态、风险债务和作战队列一起带出去。用户不依赖会员、后端接口或第三方工具，也能得到一份可保存、可粘贴、可复盘的 Markdown 闭环报告。

## 目标

- 在 `buildSprintReportMarkdown` 中加入“今日计划闭环”和“今日作战简报”两个章节。
- 复用已有 `buildDailyPlanCompletion` 和 `buildDailyPlanBrief`，不重复实现今日计划算法。
- 报告中显示完成率、已掌握数量、复习债、薄弱题、今日模拟面试次数和待办动作。
- 报告中列出今日作战队列的关键题目、来源标签、行动标签、原因、分类和题目 ID。
- 在“下一步行动”中补充今日闭环主动作，让复制出去的报告也能指导下一步。
- 保持纯前端、本地计算、完全免费，不新增接口和持久化字段。

## 非目标

- 不改复制按钮交互；现有 `SprintReportActions` 继续调用同一个 Markdown 构建函数。
- 不改今日计划生成、题目状态流转和模拟面试评分规则。
- 不导出完整题目答案，避免报告过长且偏离闭环复盘目标。
- 不引入浏览器验证；本次按用户要求只使用命令行测试、构建和 HTTP 探测。

## 设计方案

`buildSprintReportMarkdown(routes, progress, now)` 在生成健康报告和面试简报的同时，额外生成：

1. `completion = buildDailyPlanCompletion(progress, now)`
2. `dailyBrief = buildDailyPlanBrief(progress, [], now)`

`buildDailyPlanBrief` 第二个参数传空数组，依赖 `progress.questionSnapshots` 补齐题目信息。这样报告导出不需要页面额外传入题库列表，也不会触发网络请求。

新增两个渲染函数：

- `renderDailyCompletionSection(completion)`：输出今日闭环状态、摘要、完成率、核心计数和待办事项。没有待办时输出“暂无额外待办”，避免空章节。
- `renderDailyPlanBriefSection(brief)`：输出今日队列摘要和最多 8 道题，控制复制报告长度，同时保留最重要的复习债、薄弱修复和继续训练项目。

章节顺序调整为：

1. 备考健康度
2. 四维诊断
3. 今日计划闭环
4. 今日作战简报
5. 可主动表达
6. 必须规避
7. 开口热身题
8. 下一步行动

## 行为规则

- 今日计划为空时，报告仍输出“今日计划闭环”和“今日作战简报”，并给出生成今日计划的行动。
- 今日作战题目最多导出 8 条，优先级由 `buildDailyPlanBrief` 保证。
- 待办事项保留跳转路径，方便用户从外部文档回到本地产品继续处理。
- 任何章节都不能出现 `undefined`，缺失数据使用已有工具函数提供的降级文案。
- 报告内容只依赖 `StudyProgress` 和 `PrepRoute[]`，保持确定性，便于单元测试覆盖。

## 测试策略

- 先扩展 `sprintReport.test.ts`，验证非空今日计划会导出闭环状态、完成率、复习债、薄弱题、作战简报题目和今日闭环行动。
- 验证空进度报告仍包含今日闭环章节、生成计划提示，并且不出现 `undefined`。
- RED 阶段运行 `npm run test -- sprintReport`，确认测试因缺少新章节失败。
- GREEN 阶段实现最小改动后再次运行同一测试。
- 提交前运行前端全量测试、前端构建、后端测试、`git diff --check` 和本地 HTTP 200 探测。
