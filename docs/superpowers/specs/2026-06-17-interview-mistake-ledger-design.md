# 面试错因本设计

## 背景

当前练习页已经能给出单题评分、追问加压和答案差距校准，学习计划页也有面试前冲刺简报。但用户完成多次模拟面试后，仍缺少一个“长期错因本”：哪些表达维度反复低分、哪些薄弱题还没有开口练过、下一轮应该集中练哪组题。

## 目标

- 基于本地历史模拟面试记录生成错因本。
- 聚合低分维度：知识覆盖、表达结构、场景细节、边界风险。
- 将薄弱但未开口练过的题目纳入错因本。
- 为每个错因生成专项训练入口，直接跳转到 `/practice?queue=...`。
- 完全免费、本地计算、不新增后端接口。

## 设计方案

新增 `buildInterviewMistakeLedger(progress)` 工具函数：

- 遍历 `progress.interviewAttempts`，收集每次反馈中低于 70 分的维度。
- 按维度聚合平均分、出现次数、关联题目和最新题目标题。
- 遍历 `questionStates`，找出状态为 `weak` 且没有模拟面试记录的题，形成“薄弱题待开口”错因。
- 如果没有明显错因但已有练习记录，生成“进阶压力测试”入口。

新增 `InterviewMistakeLedgerPanel`：

- 放在学习计划页的面试简报之后。
- 展示主错因、错因卡片和专项训练按钮。
- 空状态提示先完成一次模拟面试。

## 行为规则

- 低分维度按“出现次数多、平均分低、最近出现”排序。
- 每个错因最多保留 8 道关联题，避免 URL 过长。
- 题目 ID 去重，优先保留最近出现的题。
- 如果维度低分和薄弱题同时存在，维度低分优先。

## 测试策略

- 单元测试覆盖空记录、重复低分维度聚合、薄弱题未开口、无错因时进阶入口、训练队列去重。
- 前端全量测试和构建验证页面接入。
