# 岗位能力地图导出设计

## 背景

首页“岗位能力地图”已经能基于用户本地题目轨迹，按 Java 后端、前端、AI 应用等路线生成准备度画像。它不需要登录，也不拉取全量题库，能把用户已经看过、练过、标记过的题转换成岗位维度的能力差距。

当前问题是能力地图只能在页面里查看。用户无法把准备度分数、薄弱数、下一步题队列和每条岗位路线的判断依据带到个人复盘文档里。对于“完全免费、个性化备考”的目标，能力地图应该像每日任务、冲刺简报一样可复制、可下载、可离线跟踪。

本阶段为岗位能力地图增加 Markdown 导出，强化“本地画像 -> 下一步训练 -> 外部复盘”的闭环。

## 目标

- 导出目标岗位、生成日期、路线数量、最弱岗位和最高准备度。
- 导出每条岗位路线的准备度、状态、覆盖题数、薄弱数、学习中数、掌握数、摘要和下一步入口。
- 空画像也能导出明确行动：打开备考路线，先建立本地题目轨迹。
- 剪贴板不可用时下载 Markdown，避免浏览器权限导致导出失败。
- 保持所有逻辑前端本地计算，不新增后端、不上传用户轨迹。

## 非目标

- 不改变 `buildAbilityMap` 的排序、打分和短板队列算法。
- 不导出完整题目答案，只导出岗位能力画像和下一步入口。
- 不新增账号、云同步或服务端报告。

## 内容结构

```markdown
# Java 后端 岗位能力地图

生成时间：2026-06-18

## 总览
- 路线数：4 条
- 最弱岗位：Java 后端，25 分
- 最高准备度：AI 应用工程师，80 分

## 岗位画像
1. Java 后端
   - 路线：Java 后端冲刺路线
   - 状态：短板明显
   - 准备度：25 分
   - 覆盖：2 道
   - 薄弱：1 道
   - 学习中：0 道
   - 掌握：1 道
   - 摘要：短板明显...
   - 下一步：/practice?queue=2
```

空画像也导出行动：

```markdown
## 岗位画像
1. Java 后端
   - 状态：待建立
   - 覆盖：0 道
   - 下一步：/routes
```

## UI 设计

- 在 `AbilityMapPanel` 标题右侧增加“复制地图”按钮。
- 使用 `CopyOutlined` 图标，和其他导出入口保持一致。
- 成功复制提示“岗位能力地图已复制”。
- 剪贴板不可用时下载 `{岗位}-岗位能力地图.md`。
- 移动端按钮左对齐，避免挤压标题和说明文案。

## 测试策略

- `abilityMap.test.ts` 增加 Markdown 导出测试：
  - 有本地题目轨迹时导出标题、日期、总览、岗位画像、准备度和训练入口。
  - 空画像导出 `/routes` 下一步，且不出现 `undefined`。
- UI 复制逻辑复用同类面板既有模式，构建和全量测试覆盖类型、引用和样式。
