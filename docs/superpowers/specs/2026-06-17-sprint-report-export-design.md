# 面试冲刺报告导出设计

## 背景

当前系统已经能在本地生成今日任务、备考健康雷达和面试前冲刺简报。用户在临近面试时仍需要把这些诊断结果带出系统：复制到自己的笔记、发给同伴复盘，或在面试前离线查看。该能力如果依赖登录、云端报告或付费导出，会削弱“完全免费”的核心承诺。

## 目标

- 基于本地学习进度生成 Markdown 格式的面试冲刺报告。
- 报告包含健康雷达、面试简报、优势、风险、热身题和下一步行动。
- 在学习计划页提供一键复制报告入口。
- 不新增后端接口、不保存云端数据、不引入付费导出或第三方服务。

## 设计方案

新增 `buildSprintReportMarkdown(routes, progress, now)` 工具函数：

- 复用 `buildPrepHealthReport` 生成整体健康度和四维诊断。
- 复用 `buildInterviewBrief` 生成优势、风险和热身题。
- 输出稳定的 Markdown 文本，便于测试、复制、版本对比和用户二次编辑。

新增 `SprintReportActions` 组件：

- 接收 `progress`。
- 生成 Markdown 报告。
- 点击按钮后优先使用 `navigator.clipboard.writeText`。
- 如果浏览器不支持剪贴板 API，则下载 `.md` 文件作为降级。


## 降级策略

复制失败或剪贴板 API 不存在时自动下载 Markdown 文件。这样即使浏览器权限收紧，用户仍能拿到报告，避免“导出按钮点了没反应”。

## 测试策略

- 单元测试验证 Markdown 包含目标岗位、健康度、最大风险、优势、风险和热身题。
- 单元测试验证空状态报告仍有明确启动行动。
- 前端全量测试和构建验证组件集成。
