# 下一轮训练队列设计

## 背景

当前系统已经有今日计划、复习排期、模拟面试错因、评分影响和多个可复制报告，但用户仍需要在多个面板之间判断“下一轮到底先练哪几题”。要真正超过普通题库，需要把这些信号收敛成一个个性化、可解释、可一键进入练习的队列。

## 目标

- 汇总当天评分影响、到期复习债、面试错因、薄弱题、学习中题和今日计划剩余题。
- 输出去重后的下一轮训练题单，每题说明来源、原因、行动和入口。
- 队列提供主行动，一键进入 `/practice?queue=...`。
- 支持 Markdown 导出，方便用户把下一轮训练题单带走复盘。
- 接入学习计划页，成为今日闭环后的直接执行面板。

## 排序规则

同一道题可能同时出现在多个来源中，只保留优先级最高的一条：

1. 今日评分影响：低分补弱、学习中复盘、高分沉淀。
2. 到期或逾期复习债：防止遗忘继续扩大。
3. 模拟面试错因：低分维度关联题优先回炉。
4. 薄弱题：还没有被前面覆盖的薄弱题。
5. 学习中题：继续巩固。
6. 今日计划剩余题：保障计划执行。

## 输出

工具层输出 `NextTrainingQueue`：

- `title`：队列状态。
- `summary`：为什么这样排。
- `metrics`：队列数、紧急项、薄弱项、面试修复项。
- `items`：题目级队列。
- `primaryAction`：一键进入下一轮训练。

页面面板展示：

- 队列标题和摘要。
- 指标卡。
- 主行动和复制按钮。
- 可点击题目列表。

## 非目标

- 不新增后端接口。
- 不修改模拟面试评分算法。
- 不替换已有今日计划、复习排期和错因面板。

## 验收

- 工具能按优先级去重并生成下一轮训练队列。
- 当天低分评分影响优先于普通复习债。
- 面试错因关联题能进入队列。
- 空进度时仍给出“先完成一次模拟面试”的行动。
- 学习计划页展示队列面板并支持复制 Markdown。
- `npm run test` 与 `npm run build` 通过。
