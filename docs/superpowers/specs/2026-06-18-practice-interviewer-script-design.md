# 模拟面试本题面试官脚本设计

## 背景

`/practice` 已经支持答题脚手架、实时结构检查、结构修复、评分、追问训练和本题重答验收。当前短板是：用户完成一次或多次评分后，页面能告诉他“哪里弱”，但还不能把最近评分自动组织成一套连续追问顺序。真实面试里，面试官不会只问一个孤立问题，而是沿着“确认基础 -> 追场景 -> 压边界 -> 做取舍”不断加压。

本阶段增加“本题面试官脚本”，在当前题侧栏根据最近评分和本题历史尝试生成 3 个连续追问步骤。它让用户不用付费、不依赖外部服务，也能获得可执行的模拟面试官流程。

## 目标

- 在 `/practice` 侧栏展示“本题面试官脚本”。
- 无评分记录时生成预热脚本，先引导用户完成一次正式回答。
- 有评分记录时基于最近一次反馈生成 3 个追问步骤：
  - 优先补最低评分维度。
  - 高分题进入方案权衡和压缩表达。
  - 回落题优先追问回落最明显维度。
- 每个追问步骤包含：
  - 追问标题
  - 面试官问题
  - 压力点
  - 回答提示
  - 预计时长
- 点击单个追问可把问题带入回答框。
- 所有逻辑前端本地完成，完全免费，不调用外部接口。

## 非目标

- 不替换现有 `FollowUpDrillPanel`。现有面板仍用于提交当前评分后的追问题单。
- 不新增语音、计时器或录音功能。
- 不修改后端评分算法。
- 不修改 `StudyProgress` 存储结构。

## 规则

### 状态

- `warmup`：当前题没有历史评分，生成首次回答预热脚本。
- `repair`：最近分数低于 60，或最低维度低于 60，生成修复型脚本。
- `pressure`：最近分数 60 到 79，生成过线加压脚本。
- `advanced`：最近分数达到 80，生成进阶追问脚本。
- `regression`：最近一次比上一次低 5 分及以上，优先进入回落修复脚本。

### 步骤排序

- 无评分：结论表达、机制说明、项目场景。
- 修复：最低维度、结构重组、边界兜底。
- 过线：最低维度、项目验证、面试官反问。
- 进阶：方案对比、风险边界、45 秒压缩表达。
- 回落：下降最多维度、丢分原因复盘、重新作答。

### 维度映射

- `coverage`：追定义、核心机制、替代方案。
- `structure`：追结论顺序、层次、时间压缩。
- `specificity`：追真实项目、数据指标、验证路径。
- `risk`：追失败边界、线上风险、兜底方案。

## UI 设计

- 面板放在 `/practice` 右侧栏，位于“本题重答验收”后。
- 标题为“本题面试官脚本”。
- 顶部显示脚本状态和总时长。
- 使用 3 个紧凑步骤卡片展示连续追问。
- 每个步骤提供“带入回答框”按钮。
- 面板布局沿用右侧栏已有密度，避免营销式大卡片。

## 测试策略

- 新增 `practiceInterviewerScript.test.ts`：
  - 无记录时输出 `warmup` 和首次回答脚本。
  - 低分最近记录输出 `repair`，首个步骤聚焦最低维度。
  - 高分记录输出 `advanced`，包含方案对比和 45 秒压缩表达。
  - 最近分回落输出 `regression`，主步骤聚焦下降最多维度。
  - 输出不包含 `undefined`。
- 新增 `PracticeInterviewerScriptPanel.test.tsx`：
  - 渲染脚本标题、状态和步骤。
  - 点击步骤按钮后回调收到追问 prompt。
