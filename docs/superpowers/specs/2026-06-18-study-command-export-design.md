# 备考指挥中心导出设计

## 背景

首页的备考指挥中心已经能聚合用户当前目标岗位、备考就绪分、最大短板、四个评分因子和下一步行动。它是用户进入平台后判断“现在最该做什么”的总入口，比只展示题库列表更接近个性化教练。

当前指挥中心只能在页面内查看。用户如果想把每日总览带到外部打卡、复盘文档、同伴监督或离线计划中，需要手动复制分数、短板和行动按钮。为了继续强化完全免费、可沉淀、可复盘的备考闭环，本次给备考指挥中心新增 Markdown 导出能力。

## 目标

- 导出目标岗位、冲刺天数、生成日期和就绪分。
- 导出指挥中心标题、摘要和最大短板。
- 导出四个评分因子：掌握沉淀、薄弱控制、表达训练、今日推进。
- 导出下一步行动列表，包含行动说明和站内路径。
- 新用户或空数据状态仍输出可执行建议，不出现 `undefined`。
- 指挥中心提供“复制指挥”按钮，剪贴板不可用时降级下载 Markdown。

## 非目标

- 不修改 `buildStudyStrategy` 的评分权重、风险判断和行动路由。
- 不新增后端接口、账号同步或云端报告。
- 不替代已有每日任务、能力地图、健康雷达等更细粒度报告。
- 不打开浏览器做视觉 QA。

## 内容结构

```markdown
# Java 后端 备考指挥中心

生成时间：2026-06-18

## 指挥概览
- 冲刺周期：21 天
- 就绪分：68
- 状态：进入可面试准备区
- 最大短板：缺少开口表达样本
- 说明：题会看不等于面试会说...

## 就绪因子
1. 掌握沉淀：3/5
   - 已掌握题越多，面试可复用素材越稳定。

## 下一步行动
1. 开始模拟面试
   - 原因：补齐开口表达样本
   - 路径：/practice
```

## UI 设计

- 在 `StudyCommandCenter` 的主卡片标题区右侧加入“复制指挥”按钮。
- 使用 `CopyOutlined` 图标，与现有导出按钮保持一致。
- 成功提示“备考指挥中心已复制”。
- 剪贴板不可用时下载 `{目标岗位}-备考指挥中心.md`。
- 移动端标题区改为网格后，按钮靠左显示，避免压缩标题和短板说明。

## 测试策略

- `studyStrategy.test.ts` 新增 Markdown 导出测试：
  - 有计划、有掌握题、有面试记录时，导出概览、因子、行动和路径。
  - 空进度仍导出启动建议，不出现 `undefined`。
- 新增 `StudyCommandCenter.test.tsx`：
  - 点击“复制指挥”会写入剪贴板。
  - 剪贴板内容包含备考指挥中心、指挥概览、就绪因子和下一步行动。
