# 本轮模拟面试补弱动作清单设计

## 背景

本轮模拟面试战报已经能判断“推进中 / 需补弱 / 已通过”，并给出平均分、通过数、最弱项和主操作。当前缺口是：用户看到“需补弱”后，只知道要重练哪些题，但不知道每道题应该先补哪个表达维度、用什么动作补。

## 目标

- 在 `PracticeSessionReport` 中增加本轮补弱动作清单。
- 每条动作绑定一道题，说明触发原因、最弱维度、建议动作和跳转地址。
- 战报面板展示前 3 条动作，保持侧栏密度可读。
- Markdown 战报导出包含完整补弱动作，方便用户粘贴到复盘笔记。

## 非目标

- 不新增后端接口。
- 不修改评分算法。
- 不改变现有主操作跳转逻辑。
- 不使用浏览器自动化验证。

## 用户价值

- 低分题不再只是“重练”，而是明确“先补结构 / 场景 / 风险 / 覆盖”。
- 整轮战报从指标看板升级成可执行复盘清单。
- 继续保持本地免费能力，不依赖付费 AI 或会员权益。

## 规则设计

- 对低于通过线的题、用户标记薄弱的题生成动作。
- 如果题目有最近一次评分，取该评分里最低的维度作为修复重点。
- 如果题目没有评分但被标弱，生成“先完成一次模拟评分”的动作。
- 动作文案需要稳定、短句、可直接执行。
- 跳转地址使用 `/practice?question={id}`，让用户回到单题模拟面试。

## 验收标准

- 风险战报返回至少一条 `repairActions`，包含题目标题、维度、原因、动作和跳转。
- 标弱但未评分的题也会出现在动作清单里。
- 战报面板渲染“本轮补弱动作”和动作条目。
- Markdown 导出包含“## 补弱动作清单”。
- `npm run test -- practiceSessionReport PracticeSessionReportPanel` 通过。
- `npm run build` 通过。
