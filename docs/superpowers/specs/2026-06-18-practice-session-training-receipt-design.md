# 模拟面试战报训练回执模板设计

## 背景

战报已经具备训练契约、训练日程和打卡清单，但用户完成打卡后仍需要把“我到底完成了什么、证据是什么、下一步做什么”写下来。没有回执模板时，闭环容易停在页面内，无法转成可复盘、可复制、可追踪的成果。

## 目标

新增“训练回执模板”，在战报中给出完成训练后应填写的四类信息：

- 训练目标：本轮要修复或推进的目标。
- 完成证据：用户需要粘贴或描述的实际完成结果。
- 阻断项：如果没有通过，明确阻断原因。
- 下一步：根据当前状态继续修复或进入下一轮。

## 非目标

- 不新增持久化回执状态。
- 不新增后端接口。
- 不做提醒、日历或站外同步。

## 方案

新增纯前端派生模型 `PracticeSessionTrainingReceipt`。它基于训练契约、训练日程和打卡清单生成 4 条回执项，每条包含填写提示和示例文本。空队列时提示先完成一次模拟面试；修复态强调阻断与修复证据；推进态强调通过证据和下一轮准备。

## 界面

在“训练日程打卡清单”之后、“下一轮训练”之前展示回执模板。卡片采用紧凑两列布局，移动端单列。主按钮为“填写训练回执”，跳转到当前训练入口。

## 测试

- Markdown 导出包含“训练回执模板”“训练目标”“完成证据”“阻断项”“下一步”。
- 空队列 Markdown 包含“等待生成训练回执”。
- 组件测试覆盖回执面板、四项字段和主按钮跳转。
