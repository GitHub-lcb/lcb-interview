# 模拟面试战报首题回执模板设计

## 背景

首题预演和验收尺已经把“怎么开始”和“怎么判断达标”讲清，但用户完成首题后还需要把结果写下来。没有回执时，首题样本很容易停留在口头完成，下一轮战报也缺少可追溯依据。

## 目标

在“首题验收尺”和“下一轮训练建议”之间增加“首题回执模板”，引导用户记录本次动作、评分证据、是否达标和下一步入口。能力全部由本地规则派生，不依赖付费服务。

## 方案

首题回执模板由 `buildPracticeSessionFirstQuestionRubric` 派生：

- 验收尺为空时显示等待态。
- 回修态生成“回修首题回执”，要求说明阻断项是否减少。
- 推进态生成“下一轮首题回执”，要求说明首题是否形成可评分样本。
- 固定 4 个字段，避免战报过长：本次动作、评分证据、是否达标、下一步。
- 主行动沿用验收尺入口，用户可以回到同一题或回修入口完成填写。

## 验收

- Markdown 导出包含“首题回执模板”，且位于“首题验收尺”之后。
- 面板展示 4 个回执字段，并提供主按钮跳转。
- 空会话显示等待态且不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
