# 追问加压训练导出设计

## 背景

模拟面试练习页已经能在用户提交回答后生成“追问加压训练”，把低分维度、面试官追问、场景压力点和答题引导组合成一组可继续训练的卡片。这个能力比单纯看题库更接近真实面试，因为它要求用户在已经回答过一次之后继续承受追问。

当前问题是追问卡片只留在页面内。用户如果想把这轮追问带到外部笔记、复盘模板、群内共练或离线训练材料，需要手动复制每张卡片。为了继续强化“完全免费、个性化备考、模拟面试闭环”的方向，本次给追问加压训练新增 Markdown 导出，让每次回答后的追问压力包可以沉淀为可复用训练材料。

## 目标

- 导出题目标题、分类、难度、生成日期、当前面试评分和等级。
- 导出追问包标题、总体说明和最多 5 个追问卡片。
- 每个追问卡片包含追问、关联维度、优先级、压力点和答题引导。
- 导出当前用户回答摘要，便于复盘“原回答暴露了什么缺口”。
- 空回答或缺少字段时仍输出可执行提示，不出现 `undefined`。
- 追问面板提供“复制追问”按钮，复制失败时降级下载 Markdown。

## 非目标

- 不改变 `buildFollowUpDrillPack` 的排序、去重和生成策略。
- 不新增后端接口、云同步、账号体系或付费能力。
- 不把标准答案全文混入导出内容，避免追问训练变成答案搬运。
- 不打开浏览器做视觉 QA，遵守当前会话的 no web browser 要求。

## 内容结构

```markdown
# HashMap 为什么线程不安全？追问加压训练

生成时间：2026-06-18

## 题目上下文
- 分类：Java 集合
- 难度：HARD
- 当前评分：55
- 当前等级：需要补强
- 当前回答摘要：HashMap 多线程不安全...

## 训练概览
追问加压训练

按最低分维度生成追问，先补最容易被面试官继续追打的缺口。

## 加压题单
1. 放到线上高并发写入场景，你会怎么选型和验证？
   - 维度：场景细节
   - 优先级：100
   - 压力点：面试官在追项目场景、验证路径和真实经验。
   - 答题引导：沿用原答案结论，补一个项目或线上场景、数据指标和验证步骤。
```

## UI 设计

- 在 `FollowUpDrillPanel` 顶部右侧加入“复制追问”按钮，和现有 `ThunderboltOutlined` 保持同组。
- 使用 `CopyOutlined` 图标，延续现有导出面板的交互语言。
- 成功提示“追问训练包已复制”。
- 剪贴板不可用时下载 `{题目标题}-追问加压训练.md`，并提示已降级下载。
- 移动端按钮换行后仍靠左显示，不压缩标题和说明。

## 测试策略

  - 空回答仍输出“未填写本轮回答”，且不出现 `undefined`。

  - 保留“带入回答框”按钮行为。
