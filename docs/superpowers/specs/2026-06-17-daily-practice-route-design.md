# 今日队列训练入口设计

## 背景

学习计划页顶部有“开始训练”按钮，但当前只是跳转到 `/practice`。当用户已经生成今日计划时，这个入口没有携带 `dailyPlan` 队列，用户进入训练页后仍可能落到默认候选题，割裂了“补齐计划 -> 理解原因 -> 开始训练”的闭环。

## 目标

- 有今日计划时，学习计划页顶部“开始训练”直接跳转到 `/practice?queue=...`。
- 队列去重，最多携带 12 道题，沿用现有训练页队列参数。
- 无今日计划时仍跳转 `/practice`。
- 抽成纯函数，避免多个入口手写 query 拼接。

## 非目标

- 不改训练页队列解析逻辑。
- 不改变今日计划生成规则。
- 不强制用户必须按今日计划训练。

## 设计方案

新增 `buildDailyPracticePath(questionIds, limit = 12)`：

1. 过滤非数字 ID。
2. 去重并截断到 `limit`。
3. 如果结果为空，返回 `/practice`。
4. 否则返回 `/practice?queue=${ids.join(',')}`。

学习计划页使用该函数：

- 顶部“开始训练”按钮调用 `navigate(buildDailyPracticePath(progress.dailyPlan))`。
- 其他已有入口暂不强行重构，避免扩大变更范围。

## 测试策略

- 单元测试覆盖空队列、去重、截断和过滤非法 ID。
- 提交前运行前端全量测试、前端构建、后端测试和 `git diff --check`。
