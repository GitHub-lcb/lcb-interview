# 模拟面试战报补弱重答模板设计

## 背景

本轮模拟面试战报已经能识别低分题和薄弱题，并给出补弱动作清单。但用户点击“去补弱”后仍需要自己把战报里的维度、原因和行动目标转写到答案框，报告到练习之间还有一次手工转换。

## 目标

- 从战报补弱动作直接切换到对应题目。
- 自动把补弱维度、低分原因和重答目标带入答案框。
- 模板必须保持免费、本地可用，不依赖新增后端接口。
- 保持现有“打开答案”“追问训练”“脚本训练”等入口不受影响。

## 交互

1. 用户在练习页右侧看到“本轮补弱动作”。
2. 点击某条动作的“去补弱”。
3. 若题目在当前队列中，页面切到该题，并把重答模板写入答案框。
4. 若题目暂不在当前队列中，保持已有跳转入口兜底到 `/practice?question=...`。

## 模板内容

模板采用面试可直接朗读的结构：

- 补弱题目
- 补弱维度
- 补弱原因
- 本次目标
- 我的重答
- 结论
- 原因
- 场景
- 边界

不同低分维度可补充不同提示，例如覆盖度强调定义、机制和替代方案，结构化强调结论到边界，场景细节强调项目触发条件和指标，风险意识强调失败边界和兜底方案。

## 验收

- 工具层能为补弱动作生成稳定模板。
- 面板点击补弱动作会触发模板使用回调。
- 练习页能切换到目标题并保留模板，不被题目切换清空逻辑覆盖。
- `npm run test` 与 `npm run build` 通过。
