# 模拟面试战报首题复用复盘回流清单设计

## 背景

首题复用复盘归档包已经能把复用后的分数、证据、阻断和下一题种子保存成可读材料，但归档材料仍偏“看完留存”。用户真正进入下一轮训练时，还需要把这些材料重新拆成动作：先看哪项分数、带入哪段证据、遇到阻断怎么处理、下一题从哪里启动。如果缺少回流清单，首题复用链路会停在归档层，不能稳定变成下一次训练的第一步。

## 目标

在“首题复用复盘归档包”和“下一轮训练建议”之间增加“首题复用复盘回流清单”。清单把归档包转换成 4 个下一轮可执行动作：分数回流、证据带入、阻断处置、下一题启动。所有内容继续由本地规则从当前战报上下文派生，不依赖后端状态、账号体系、付费 AI 或第三方服务。

## 方案

首题复用复盘回流清单由 `buildPracticeSessionFirstQuestionReuseReviewArchive` 派生：

- 归档包为空时，显示“等待首题复用复盘回流”，提示先生成归档包。
- 归档包处于 repair 状态时，显示“回修复用复盘回流清单”，强调先补分数、补证据和处理阻断。
- 归档包处于 ready 状态时，显示“首题复用复盘回流清单”，强调把归档内容直接带入下一轮首题动作。
- 每个动作包含执行说明、开场提示、验收规则和回退动作，用户可以逐项照着进入下一轮。
- 主行动复用归档包的入口，避免新增路由、持久化状态或复杂交互。

## 用户价值

相比只展示归档结果，回流清单让用户知道“下一轮第一分钟该做什么”。它把免费本地模拟面试从记录型报告推进到动作型闭环：上一轮的复用复盘不会沉没在文档里，而是直接驱动下一轮训练的开场、验收和失败回退。

## 验收

- Markdown 导出包含“首题复用复盘回流清单”，且位于“首题复用复盘归档包”之后、“下一轮训练建议”之前。
- 前端战报展示 4 个回流动作：分数回流、证据带入、阻断处置、下一题启动。
- 每个回流动作展示“开场提示”“验收规则”“回退动作”。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
