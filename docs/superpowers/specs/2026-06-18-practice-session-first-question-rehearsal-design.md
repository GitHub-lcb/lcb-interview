# 模拟面试战报首题预演卡设计

## 背景

战报已经能从本轮复盘推进到准入闸门、启动包和启动执行清单，但用户真正进入下一轮训练前，还缺少一个“第一题怎么开口”的明确预演。只展示下一轮题目列表会让用户重新判断为什么先做这题、开场应该说什么、什么算通过。

## 目标

在“启动执行清单”和“下一轮训练建议”之间增加“首题预演卡”，把下一轮训练队列的第一道题转成可执行的面试开场卡。用户在不付费、不依赖外部服务的前提下，直接看到首题标题、选择原因、开场提示、通过信号和必须留下的证据。

## 方案

首题预演卡完全由前端派生，不新增后端接口和持久化字段：

- 复用 `buildPracticeSessionLaunchChecklist` 判断当前是否已具备启动上下文。
- 复用 `buildPracticeSessionNextTrainingQueue(..., limit=5)` 获取下一轮首题。
- 当启动清单为空或下一轮队列为空时显示等待态，提示先完成模拟面试并生成启动执行清单。
- 当准入仍处于回修态时，卡片进入 `repair` 状态，首题改为首个回修动作，避免用户带着阻断项进入下一轮。
- 当准入通过时，卡片进入 `ready` 状态，展示下一轮第一题的开场预演。

## 展示内容

- 标题：`等待首题预演`、`回修首题预演`、`下一轮首题预演`。
- 首题名称：回修态取启动清单首项，推进态取下一轮队列第一题。
- 选择原因：说明为什么先处理它。
- 开场提示：给出一句开口作答的起手式。
- 通过信号：说明完成后应满足什么条件。
- 证据要求：说明必须留下的评分样本或回修证据。
- 主行动：进入对应题目或回修入口。

## 验收

- Markdown 战报在“启动执行清单”之后、“下一轮训练建议”之前导出首题预演卡。
- 前端面板同样按这个顺序展示，并且点击主按钮进入首题或回修入口。
- 空会话不出现 `undefined`。
- 聚焦测试、全量测试和构建通过。
