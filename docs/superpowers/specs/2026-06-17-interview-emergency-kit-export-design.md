# 面试前急救包导出设计

## 背景

面试前急救包已经能把本地学习进度、复习债、错因、薄弱题、今日闭环和模拟样本压缩成 30 分钟内的行动队列。它解决的是“临近面试时先做什么”的问题，但当前结果只能在学习计划页查看，用户无法把最后一轮行动清单复制到手机、笔记、候选人准备文档或离线面试脚本里。

本阶段把急救包导出为 Markdown。导出能力保持完全前端、本地、免费，不新增账号、后端接口或云端存储，让用户在没有会员和网络依赖的情况下也能带走临场清单。

## 目标

- 新增 `buildInterviewEmergencyKitMarkdown(progress, now)` 纯函数，复用现有 `buildInterviewEmergencyKit(progress, now)`。
- Markdown 覆盖目标岗位、生成日期、急救概览、指标、行动列表、关联题目和入口。
- 在 `InterviewEmergencyKitPanel` 增加“复制急救包”按钮。
- 剪贴板不可用时降级下载 Markdown 文件，避免浏览器权限导致导出失败。
- 增加工具函数和组件交互测试，覆盖高风险急救包和空状态导出。

## 非目标

- 不修改急救包排序、耗时预算和风险等级算法。
- 不新增后端接口、数据库表或远程同步。
- 不导出完整题库答案，只导出当前用户本地计算出的最后一轮行动清单。
- 不使用浏览器 QA；本轮按用户要求只用命令行测试、构建和 HTTP 探测。

## 方案

### 方案 A：复制面板可见文本

实现最少，但会丢失入口、关联题目、生成时间和空状态动作，也难以单测结构稳定性。

### 方案 B：新增 Markdown 构建纯函数并由面板调用

复用现有急救包模型，把页面展示与可携带文本分离。纯函数可用 Vitest 覆盖结构和空状态，面板只负责复制与下载降级。

### 方案 C：只依赖冲刺报告的总导出

冲刺报告适合完整复盘，但面试前急救包强调最后 30 分钟。只依赖总报告会增加操作路径，用户在临场场景下仍需要单独复制。

选择方案 B。它把急救包从“页面提醒”升级成“可携带面试前作战清单”，同时不改变现有算法边界。

## Markdown 结构

```md
# Java 后端 面试前急救包

生成时间：2026-06-17

## 急救概览
- 状态：面试前先压最高风险
- 摘要：先处理复习债和错因...
- 预计耗时：24 分钟
- 下一步：先清复习债，/practice?queue=1
- 说明：先把最容易遗忘的题目拉回可答状态。

## 指标
- 急救动作：4，今日优先完成
- 预计耗时：24 分钟，控制在 30 分钟内

## 急救动作
1. 1 道复习债先清掉
   - 类型：复习债
   - 耗时：8 分钟
   - 动作：开始复习
   - 说明：先处理已经到期或逾期的题目。
   - 原因：临场前最怕看过但忘记。
   - 题目：1
   - 入口：/practice?queue=1
```

空状态也必须导出明确动作：

```md
## 急救动作
1. 先建立一次模拟面试样本
   - 类型：模拟样本
   - 入口：/practice
```

## UI

- `InterviewEmergencyKitPanel` 的行动区域增加“复制急救包”小按钮。
- 使用 `CopyOutlined` 图标，保持 Ant Design 按钮风格。
- 成功复制提示“面试急救包已复制”。
- 剪贴板失败时下载 `{目标岗位}-面试前急救包.md`，并提示“剪贴板不可用，已下载 Markdown 急救包”。
- 不新增大面积布局和说明文案，避免急救包面板变重。
