# 模拟面试复盘导出设计

## 背景

首页和练习页已经有“免费模拟面试复盘”模块，会基于本地面试评分记录汇总表达趋势、平均分、最高分、覆盖题目数、最弱维度和最近记录。它解决的是面试题库类产品的核心短板：用户不仅要看到答案，还要知道自己能不能在真实面试中讲清楚。

当前复盘结果只能在页面内查看。用户如果想把表达趋势写进个人复盘、发送给学习搭子、沉淀到面试准备文档，需要手动摘录分数和短板维度。为了继续强化“完全免费、个性化备考、模拟面试、学习闭环”的目标，本阶段为模拟面试复盘增加 Markdown 导出，让表达能力证据可以离线保存和持续追踪。

## 目标

- 导出目标岗位、生成日期、表达趋势、平均分、最高分、最近一次分数、练习次数和覆盖题目数。
- 导出当前最弱维度，包含维度名称、平均分、样本数和摘要。
- 导出全部维度均分，方便用户看到覆盖度、结构化、具体性、风险意识的相对短板。
- 导出最近 3 条模拟面试记录，包含题目标题、得分、时间和题目入口。
- 空状态仍导出“开始模拟面试”的下一步，不出现 `undefined`。
- 面板提供“复制复盘”按钮，剪贴板不可用时降级下载 Markdown。

## 非目标

- 不改变 `buildInterviewReviewSummary` 的趋势、短板维度和推荐语算法。
- 不导出用户的原始回答全文，避免把可能包含隐私的内容带出系统。
- 不新增后端接口、账号体系或云端同步。

## 内容结构

```markdown
# Java 后端 模拟面试复盘

生成时间：2026-06-18

## 复盘概览
- 趋势：需要复盘
- 平均分：72
- 最高分：88
- 最近一次：66
- 练习次数：5
- 覆盖题目：3
- 建议：最近表现回落，先放慢节奏复盘...

## 当前短板
- 维度：具体性
- 平均分：58
- 样本数：5
- 摘要：具体性 58

## 维度均分
1. 具体性：58 分，5 次
2. 风险意识：64 分，5 次

## 最近记录
1. HashMap 为什么线程不安全？
   - 得分：66
   - 时间：2026-06-18
   - 入口：/question/1

## 下一步
- 动作：进入模拟面试
- 入口：/practice
```

## UI 设计

- 在 `InterviewReviewPanel` 标题区增加“复制复盘”按钮。
- 使用 `CopyOutlined` 图标，保持和首页其他导出按钮一致。
- 空状态和有记录状态都展示复制按钮，空状态导出下一步行动。
- 成功复制提示“模拟面试复盘已复制”。
- 剪贴板不可用时下载 `{岗位}-模拟面试复盘.md`。
- compact 模式也允许复制，因为练习页用户更接近刚完成面试后的复盘场景。

## 测试策略

- `interviewReview.test.ts` 增加 Markdown 导出测试：
  - 有记录状态导出标题、日期、概览、当前短板、维度均分、最近记录和题目入口。
  - 空状态导出开始模拟面试动作和 `/practice`。
  - 导出结果不包含 `undefined`。
- 新增或扩展 `InterviewReviewPanel.test.tsx`：
  - 点击“复制复盘”会写入剪贴板。
  - 复制内容包含复盘标题、复盘概览和入口。
