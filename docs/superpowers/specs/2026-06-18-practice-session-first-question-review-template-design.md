# 模拟面试战报首题复盘模板设计

## 背景

首题放行门禁解决了“能不能进入下一轮”的裁决问题，但用户进入下一轮并完成第一题后，还需要知道复盘时必须留下哪些字段。如果没有复盘模板，下一轮首题可能只产生一次分数，而没有沉淀可比较的证据链。

## 目标

在“首题放行门禁”和“下一轮训练建议”之间增加“首题复盘模板”。模板把下一轮第一题作答后的复盘压缩成 4 个字段：评分变化、证据变化、阻断变化、下一题动作。所有字段由本地规则生成，继续保持完全免费使用。

## 方案

首题复盘模板由 `buildPracticeSessionFirstQuestionReleaseGate` 派生：

- 放行门禁为空时，显示等待状态。
- 放行门禁阻断时，生成回修复盘模板，要求先记录阻断项是否减少。
- 放行门禁 ready 时，生成下一轮首题复盘模板，要求记录首题是否形成新样本。
- 每个字段包含填写提示、示例和验收规则，便于用户直接复制到复盘笔记中。
- 主行动沿用放行门禁入口，避免引入新的跳转来源。

## 用户价值

相比只提示“进入下一轮”，复盘模板让用户在下一轮第一题之后立刻留下可比较证据：分数是否提升、表达证据是否增强、阻断是否减少、下一题如何推进。训练闭环从“开始下一轮”推进到“验证下一轮第一题结果”。

## 验收

- Markdown 导出包含“首题复盘模板”，且位于“首题放行门禁”之后。
- 前端战报展示 4 个复盘字段和主按钮。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
