# 面试后闭环教练设计

## 背景

练习页已经能完成模拟面试评分、追问加压和答案差距分析，但评分之后仍缺少一个明确的“现在立刻做什么”闭环。用户看到分数和建议后，可能在重答、追问、标记薄弱、查看答案和进入下一题之间犹豫，导致训练链路中断。

## 目标

- 完全免费，基于当前题目、用户答案和本地/后端评分结果生成即时闭环。
- 把一次模拟面试后的下一步压缩为最多 3 个动作。
- 动作必须能直接触发页面已有能力：带入重答提示、标记薄弱、标记掌握、打开答案、进入下一题。
- 不新增后端接口，不改变现有评分流程。

## 方案

新增 `buildPracticeFeedbackClosure(question, answer, feedback)` 纯函数，输出 `PracticeFeedbackClosure`：

- `level`：`repair`、`follow-up`、`pass`、`excellent`。
- `title` / `summary`：解释当前回答应该如何闭环。
- `metrics`：分数、最低维度、答案长度、追问数。
- `actions`：最多 3 个动作，每个动作包含 `kind`、`label`、`description`、`prompt`。

动作策略：

- 分数低于 60，或最低维度低于 60：优先重答最低维度，并建议标记薄弱。
- 分数 60-79：优先选择一个追问继续加压，并建议打开答案核对。
- 分数 80 以上：建议标记掌握，并选择进阶追问保持上限。
- 答案过短时，额外提示补项目场景和边界。

## 页面呈现

新增 `PracticeFeedbackClosurePanel`，放在评分结果之后、追问加压训练之前。面板展示闭环标题、4 个指标和最多 3 个动作按钮。按钮通过回调接入 Practice 页已有函数：

- `rewrite` / `follow-up`：调用 `startFollowUpAnswer(prompt)` 把提示带入输入框。
- `weak`：调用 `markWeak()`。
- `mastered`：调用 `markMastered()`。
- `answer`：跳到题目详情。
- `next`：进入下一题。

## 测试

- 工具函数测试覆盖低分修复、追问加压、高分掌握、短答案补场景。
- 组件测试确认面板渲染标题、指标、动作按钮，并触发回调。
- Practice 页只做轻量接入，不新增复杂页面测试。
