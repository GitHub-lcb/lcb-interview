# 模拟面试答题脚手架设计

## 背景

`/practice` 已经能把题目组织成模拟面试队列，并在用户提交回答后给出评分、追问、收口动作和答案差距。但真实面试里最大的断点通常发生在提交之前：用户看过答案，却不知道第一句话怎么开、机制和项目场景怎么串、边界风险放在哪里。

如果页面只提供空白输入框，用户会继续停留在“刷题阅读”状态；要做到完全免费且显著强于常规题库，练习页需要在回答前给出可直接口述的结构，让用户先把答案搭起来，再进入评分和追问闭环。

本阶段增加“答题脚手架”面板，为当前题生成本地、免费的结构化口述提纲，并支持一键带入回答框。

## 目标

- 在 `/practice` 当前题下方展示答题脚手架，位于正式回答输入框之前。
- 脚手架固定覆盖四段：结论、机制、场景、边界。
- 根据题目标题、分类、标签和目标岗位生成更贴近技术域的提示。
- 支持一键把完整回答模板带入输入框，帮助用户从空白回答变成可改写草稿。
- 支持复制 Markdown 脚手架；剪贴板不可用时下载 Markdown 文件。
- 纯函数输出不得出现 `undefined`，空目标岗位也要有兜底文案。

## 非目标

- 不接入新的后端接口，不调用外部 AI。
- 不替代提交后的评分、追问和答案差距分析。
- 不生成完整标准答案，只生成回答结构和口述模板。
- 不改变练习队列生成、状态流转或评分算法。

## 内容结构

脚手架由纯函数生成：

```text
标题：先搭好 4 句回答
摘要：按“结论 -> 机制 -> 场景 -> 边界”组织，提交评分前先把空回答变成可开口版本。

1. 先给结论
   - 提示：用一句话直接回答题目，不绕背景。
2. 补核心机制
   - 提示：结合 Java 集合、并发安全和扩容链路解释。
3. 带项目场景
   - 提示：放到 Java 后端项目里说明触发条件、处理动作和验证指标。
4. 收边界风险
   - 提示：说明替代方案、失败模式和常见误区。
```

带入回答框的模板：

```markdown
结论：
我会先说明「HashMap 为什么线程不安全？」的核心结论：...

机制：
这里的关键机制是...

场景：
在 Java 后端项目里，我会把它放到...

边界：
需要注意的是...
```

## 领域提示策略

- Java / JVM / Spring：强调运行机制、并发边界、生命周期和工程取舍。
- MySQL / Redis / MongoDB / Elasticsearch：强调数据结构、事务一致性、索引、缓存和性能观测。
- 消息队列 / Kafka / RabbitMQ：强调可靠性、顺序性、幂等、积压和补偿。
- 系统设计 / 运维 / DevOps：强调容量、可用性、故障隔离、监控和回滚。
- 前端 / React / Vue / JavaScript / TypeScript：强调渲染链路、状态边界、性能和兼容性。
- AI 大模型 / AI 项目：强调 RAG、评测、成本、幻觉和权限边界。
- 未命中特定领域时使用通用结构提示。

## UI 设计

- 面板放在题目卡片和回答输入框之间，作为“提交前准备”。
- 标题区域包含 `FormOutlined` 图标、标题、摘要、复制按钮和“带入回答框”主按钮。
- 四个结构点用紧凑网格展示，避免占用过高首屏。
- 移动端单列展示，按钮自动换行。
- 成功复制提示“答题脚手架已复制”。
- 剪贴板不可用时下载 `{题目标题}-答题脚手架.md`，并提示“剪贴板不可用，已下载 Markdown 脚手架”。
