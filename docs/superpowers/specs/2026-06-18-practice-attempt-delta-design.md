# 模拟面试本题重答验收设计

## 背景

`/practice` 现在已经支持答题脚手架、实时结构检查、结构缺口修复、提交评分、追问和答案差距校准。它已经能帮助用户完成一次模拟面试闭环，但同一题多次重答后，页面只显示最近分数，用户无法马上判断“这次修复到底有没有变好”。

真实备考中，重答的价值不在于多刷一次，而在于确认分数和短板维度是否改善。本阶段增加“本题重答验收”，把同一题最近两次模拟评分做对比，直接显示总分变化、维度变化和下一次重答提示。

## 目标

- 在 `/practice` 侧栏展示当前题的重答验收面板。
- 无记录时提示先完成一次模拟评分。
- 只有一次记录时显示基线分和“再答一次验收”动作。
- 至少两次记录时比较最近一次和上一次：
  - 总分变化
  - 覆盖度、结构、场景、风险四个维度变化
  - 趋势：进步、回落、稳定
- 生成一个可带入回答框的下一次重答提示。
- 所有逻辑前端本地完成，完全免费，不调用外部接口。

## 非目标

- 不改 `recordInterviewAttempt` 的存储结构。
- 不改正式评分算法。
- 不展示超过最近两次的复杂折线图。
- 不强制用户必须重答才能进入下一题。

## 规则

### 状态

- `empty`：当前题没有模拟记录。
- `single`：只有一次记录，建立基线。
- `improved`：最近分数比上一次高 5 分及以上。
- `regressed`：最近分数比上一次低 5 分及以上。
- `stable`：分数变化在 -4 到 +4 之间。

### 维度对比

按评分维度 `coverage`、`structure`、`specificity`、`risk` 对齐。若某次缺少某个维度，则该维度用 0 兜底，避免输出 `undefined`。

优先动作：

- 无记录：`先完成一次模拟评分`
- 单次记录：`再答一次验收`
- 回落：按下降最多的维度重答
- 稳定：按当前最低维度重答
- 进步但未到 80 分：继续补当前最低维度
- 进步且达到 80 分：提示进入下一题或继续追问

## UI 设计

- 面板放在 `/practice` 右侧栏，位于“本轮概览”后。
- 标题为“本题重答验收”。
- 展示最近分、上次分、变化值和趋势摘要。
- 维度变化用四个紧凑条目展示，绿色表示提升，橙色表示回落，灰色表示持平。
- 主按钮文案根据状态变化，如“再答一次验收”“按场景细节重答”。
- 点击主按钮后，把重答提示写入回答框并清空旧评分。

## 测试策略

- 新增 `practiceAttemptDelta.test.ts`：
  - 空记录输出 `empty`，提示先完成模拟评分。
  - 单次记录输出 `single`，主动作包含重答提示。
  - 最近分数提升时输出 `improved`、正向分差和维度差。
  - 最近分数回落时输出 `regressed`，主动作聚焦下降最多维度。
  - 输出不包含 `undefined`。
- 新增 `PracticeAttemptDeltaPanel.test.tsx`：
  - 两次记录时渲染趋势、分差和维度变化。
  - 点击主按钮后回调收到重答提示。
