# 备考指挥中心下一轮训练设计

## 背景

首页备考指挥中心已经能根据本地学习进度给出就绪分、最大短板、就绪因子和下一步行动。它解决“现在状态如何”的问题，但还没有直接回答“下一轮到底练哪几题”。学习计划页已有下一轮训练队列，工作台日报和冲刺报告也已接入导出，因此首页指挥中心应该复用同一套队列，成为用户进站后的第一训练入口。

## 目标

- 指挥中心页面内展示下一轮训练摘要、主行动和前三个候选题。
- 指挥中心 Markdown 导出增加 `## 下一轮训练队列`，便于外部复盘文档保留训练入口。
- 全部复用 `buildNextTrainingQueue`，不复制评分、复习债、错因和题目状态排序逻辑。
- 空状态仍可执行：没有队列时引导用户先做一次模拟面试或生成今日计划。

## 非目标

- 不改下一轮训练队列本身的排序算法。
- 不新增后端接口、数据库表或账号能力。
- 不把首页变成完整训练页；这里只放摘要和入口，详细队列仍在学习计划页。

## 页面行为

`StudyCommandCenter` 在就绪因子下方、行动按钮上方增加轻量队列摘要：

- 标题：下一轮训练
- 摘要：使用 `queue.summary`
- 主按钮：使用 `queue.primaryAction`
- 列表：展示最多 3 个 `queue.items`，每项包含题目、来源、行动和状态信息。

空队列时显示一条引导文案，不展示空白卡片。

## 导出行为

`buildStudyCommandMarkdown(progress, now)` 内部构建 `buildNextTrainingQueue(progress, now, 5)`，并在 `## 就绪因子` 与 `## 下一步行动` 之间输出：

```md
## 下一轮训练队列
- 状态：下一轮训练队列
- 摘要：已按评分影响、复习债、面试错因和今日计划排好 2 道题。
- 主行动：开始下一轮训练，/practice?queue=1,2
1. Java 并发题目 1
   - 来源：评分影响
   - 行动：重答补强，入口：/practice?queue=1
```

## 验收标准

- 指挥中心组件能渲染“下一轮训练”和主行动按钮。
- 复制指挥中心 Markdown 时包含下一轮训练队列章节。
- 空状态导出包含“暂无下一轮训练题”并且不出现 `undefined`。
- `npm run test -- studyStrategy StudyCommandCenter` 通过。
- `npm run test`、`npm run build` 通过。
