# 今日作战简报导出设计

## 背景

今日作战简报已经能把本地 `dailyPlan` 拆成每道题的训练原因：复习债优先补、薄弱题优先讲清、学习中题继续巩固、新题建立首轮记忆、掌握题轻量回看。它是用户每天进入学习闭环的第一张操作清单。

当前简报只能在学习计划页查看。为了让产品在免费能力上继续拉开差距，需要让用户一键复制当天题单、原因和入口，放到备忘录、聊天窗口或离线计划里执行。本阶段新增 Markdown 导出，不改变计划生成算法，不引入后端和账号能力。

## 目标

- 新增 `buildDailyPlanBriefMarkdown(progress, candidates, now)` 纯函数，复用 `buildDailyPlanBrief(progress, candidates, now)`。
- Markdown 包含目标岗位、生成日期、概览、指标和今日题单。
- `DailyPlanBriefPanel` 增加“复制简报”按钮。
- 剪贴板不可用时下载 Markdown 文件，避免权限失败导致用户带不走计划。
- 增加工具函数和组件交互测试，覆盖有计划和空计划导出。

## 非目标

- 不修改今日计划生成、排序和复习债判断算法。
- 不新增后端接口、数据库表、登录、会员或云同步。
- 不导出完整题目答案，只导出当天训练题单和执行原因。
- 不使用浏览器 QA；继续只用命令行测试、构建和 HTTP 探测。

## 方案

### 方案 A：复制面板可见文本

实现最快，但会丢失生成日期、入口、指标细节和空状态动作，也无法稳定单测结构。

### 方案 B：新增 Markdown 构建纯函数并由面板调用

复用 `buildDailyPlanBrief` 的结构化输出，纯函数负责可携带文本，面板只负责复制和下载降级。测试覆盖导出结构与用户交互。

### 方案 C：只依赖冲刺总报告

总报告适合阶段复盘，但今日作战简报是当天执行清单。只依赖总报告会让日常学习路径过重。

选择方案 B。它直接增强每日学习闭环，且和现有导出面板保持一致。

## Markdown 结构

```md
# Java 后端 今日作战简报

生成时间：2026-06-17

## 作战概览
- 状态：今日计划已拆解
- 摘要：今日 3 道题：1 道复习债，1 道薄弱题，1 道新题。
- 题量：3 道

## 指标
- 计划题量：3 道，按今日计划顺序执行
- 复习债：1 道，优先补回记忆窗口

## 今日题单
1. HashMap 为什么线程不安全？
   - 分类：Java 并发
   - 来源：复习债
   - 状态：weak
   - 动作：先复盘
   - 原因：这道题已逾期，先补回记忆断点。
   - 入口：/question/1
```

空状态也导出明确下一步：

```md
## 今日题单
- 今日计划还未生成。先在学习计划页生成题单，再复制作战简报。
```

## UI

- `DailyPlanBriefPanel` 头部右侧增加“复制简报”小按钮。
- 保留原有“展示 6/10”或“待生成”状态。
- 使用 `CopyOutlined` 图标，成功提示“今日作战简报已复制”。
- 剪贴板失败时下载 `{目标岗位}-今日作战简报.md`。
- 头部行动区使用轻量 flex 布局，移动端左对齐。
