# 模拟面试本题脚本一键继续动作设计

## 背景

本题面试官脚本已经能展示逐问进度，也能导出进度复盘。当前用户仍需要在三张追问卡片中自行寻找下一步。对高频练习来说，这一步会打断节奏。

## 目标

- 在脚本进度区提供一个主操作按钮，直接把当前最该练的追问带入回答框。
- 已通过前置追问时，按钮指向下一道待练追问。
- 存在修复中追问时，按钮优先指向修复中的追问。
- 全部通过时不显示继续按钮，避免误导用户。

## 非目标

- 不新增后端状态。
- 不改变逐问卡片按钮；卡片仍可单独重练任意追问。
- 不使用浏览器自动化验证。

## 用户价值

- 用户在单题练习时可以顺着系统建议连续推进，不需要手动判断下一题。
- 修复中追问会被优先拉回，减少“跳过半成品回答”的问题。
- 相比只给题目列表的平台，这里提供完整的本题追问训练节奏。

## 体验规则

- 如果下一步是待练追问，按钮文案为“继续下一问”。
- 如果下一步是修复中追问，按钮文案为“修复当前问”。
- 点击按钮后调用现有 `onUsePrompt`，把对应追问带入回答框。
- “复制进度”仍为次要操作，放在同一操作区。

## 验收标准

- 第一问已通过时，点击“继续下一问”带入第二问 prompt。
- 第一问修复中时，点击“修复当前问”带入第一问 prompt。
- 三问全通过时，不显示继续按钮。
- `npm run test -- PracticeInterviewerScriptPanel` 通过。
- `npm run build` 通过。
