# 最后 24 小时面试简报导出设计

## 背景

最后 24 小时面试简报已经能根据本地学习轨迹生成进场信心、复习债、错因禁忌、主线话术和收尾动作。它覆盖的是面试前一天到进场前的准备窗口，比 30 分钟急救包更早，比冲刺报告更轻。

当前简报只能在学习计划页查看。用户在面试前一天通常会把进场清单同步到手机、备忘录或打印纸上，如果不能单独复制，就会被迫打开完整网页或导出更重的冲刺报告。本阶段把简报导出为 Markdown，让这份临场清单完全免费、离线可携带。

## 目标

- 新增 `buildInterviewLastMinuteBriefMarkdown(progress, now)` 纯函数，复用现有 `buildInterviewLastMinuteBrief(progress, now)`。
- Markdown 包含目标岗位、生成日期、简报概览、指标、进场动作和入口。
- 在 `InterviewLastMinuteBriefPanel` 增加“复制简报”按钮。
- 剪贴板不可用时下载 Markdown 文件，保证导出路径稳定。
- 增加工具函数和组件交互测试，覆盖高风险简报和空状态导出。

## 非目标

- 不修改进场信心、风险等级、动作排序和候选项生成算法。
- 不新增后端接口、登录、会员、云同步或浏览器存储结构。
- 不替代冲刺报告总导出；本能力只服务最后 24 小时单面板。
- 不使用浏览器 QA；本轮继续只用命令行测试、构建和 HTTP 探测。

## 方案

### 方案 A：复制页面可见文本

实现快，但会丢失生成时间、入口、动作分类和空状态提示，也无法稳定单测 Markdown 结构。

### 方案 B：新增 Markdown 构建纯函数并由面板调用

复用 `buildInterviewLastMinuteBrief` 的结构化结果，导出内容与页面一致但更适合离线阅读。纯函数可单测，面板只负责复制和下载降级。

### 方案 C：只依赖冲刺报告里的“最后 24 小时”章节

总报告已经包含该章节，但用户临场时需要一页轻清单。只依赖总报告会增加入口成本，弱化“最后一天打开即用”的体验。

选择方案 B。它把最后 24 小时简报从页面模块升级为独立可携带的面试前清单，同时保持所有计算逻辑本地免费。

## Markdown 结构

```md
# Java 后端 最后 24 小时面试简报

生成时间：2026-06-17

## 简报概览
- 状态：最后 24 小时先压临场风险
- 摘要：进场信心 45 分，优先处理 1 道复习债和 4 个错因信号。
- 进场信心：45 分
- 下一步：先复盘高风险题，/practice?queue=1
- 说明：先用最短路径把逾期和今日到期题过一遍...

## 指标
- 进场信心：45 分，仍需校准
- 面试样本：1 次，已有表达证据

## 进场动作
1. 1 道复习债面试前必须回看
   - 类型：必看复习
   - 动作：先复盘高风险题
   - 说明：先用最短路径把逾期和今日到期题过一遍...
   - 证据：薄弱题每天复习一次，直到能稳定讲清。
   - 题目：1
   - 入口：/practice?queue=1
```

空状态也必须导出下一步：

```md
## 进场动作
1. 先建立一题真实开口样本
   - 类型：模拟样本
   - 动作：先做一题模拟
   - 入口：/practice
```

## UI

- `InterviewLastMinuteBriefPanel` 的行动区域增加“复制简报”小按钮。
- 使用 `CopyOutlined` 图标，和其他导出按钮保持一致。
- 成功复制提示“面试简报已复制”。
- 剪贴板失败时下载 `{目标岗位}-最后24小时面试简报.md`。
- 收紧行动区说明文字 CSS，避免影响按钮内部 Ant Design 文本样式。
