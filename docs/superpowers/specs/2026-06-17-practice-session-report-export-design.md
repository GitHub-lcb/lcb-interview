# 本轮模拟面试战报导出设计

## 背景

练习页已经有“本轮模拟面试战报”，能展示当前队列的已答题数、平均分、通过数、最弱项和下一步动作。但这个战报目前只能停留在页面上，用户无法把一轮模拟面试结果复制到笔记、发给同伴复盘，或留作当日复盘记录。

为了继续强化“完全免费个人面试教练”，本阶段要把本轮战报导出为 Markdown。它不依赖登录、云端报告或付费能力，完全基于浏览器本地学习轨迹生成。

## 目标

- 新增 `buildPracticeSessionReportMarkdown(queue, progress, now)`。
- Markdown 包含本轮摘要、核心指标、题目队列、低分/薄弱题和下一步行动。
- 在 `PracticeSessionReportPanel` 增加“复制战报”按钮。
- 剪贴板不可用时下载 Markdown 文件，避免导出能力在浏览器权限受限时失效。
- 保持练习侧栏布局紧凑，不改变主流程。

## 非目标

- 不新增后端接口。
- 不修改模拟面试评分算法。
- 不新增云端历史报告。
- 不改变练习队列生成逻辑。

## 方案

### 方案 A：只在面板上增加复制当前可见文本

实现最小，但只能复制摘要和指标，缺少题目队列、低分题和下一步入口，复盘价值不足。

### 方案 B：新增 Markdown 构建函数并在面板复制

复用 `buildPracticeSessionReport(queue, progress)`，把完整战报结构化导出。UI 只增加一个命令按钮，测试可以覆盖纯函数和组件交互。

### 方案 C：新增练习页完整日报导出

范围更大，需要串联答案缺口、追问训练和单题闭环。本阶段会拉大改动面，适合后续迭代。

选择方案 B。它能用小范围改动补齐本轮练习复盘导出，并为后续整合答案缺口和单题闭环留下接口。

## Markdown 结构

```md
# Java 后端 本轮模拟面试战报

生成时间：2026-06-17
队列题数：3

## 本轮摘要
- 状态：本轮优先补弱
- 说明：已答 2/3，平均 59 分；3 道题需要先补弱重练。
- 已答：2/3
- 平均分：59
- 通过数：0
- 低分/薄弱题：1, 2, 3

## 核心指标
- 已答题：2 / 3，剩余 1 道
- 平均分：59 分，低于通过线
- 通过数：0 道，70 分以上计为通过
- 最弱项：结构化 53，结构表达一般

## 题目队列
- 1. Java 面试题 1：Java 基础，MEDIUM，来源 plan，状态 learning，最近评分 62 分
- 2. Java 面试题 2：Java 基础，MEDIUM，来源 plan，状态 learning，最近评分 55 分
- 3. Java 面试题 3：Java 基础，MEDIUM，来源 plan，状态 weak，最近评分 暂无

## 下一步行动
- 补弱重练：优先回到低分题和标弱题，先把本轮短板修到通过线。（/practice?queue=1,2,3）
```

空状态也导出：

```md
# Java 后端 本轮模拟面试战报

## 本轮摘要
- 状态：先选择一组面试题
- 说明：当前还没有练习队列，先从学习计划、弱题或题库中选择题目进入模拟面试。
```

## UI

- 在 `PracticeSessionReportPanel` 的头部状态区域增加小尺寸按钮“复制战报”。
- 按钮使用 `CopyOutlined`，和现有 Ant Design 按钮风格保持一致。
- 成功复制后提示“本轮模拟面试战报已复制”。
- 失败时下载 `目标岗位-本轮模拟面试战报.md`，并提示“剪贴板不可用，已下载 Markdown 战报”。
