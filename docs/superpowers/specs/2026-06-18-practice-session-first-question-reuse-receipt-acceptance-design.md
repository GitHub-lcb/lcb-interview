# 模拟面试战报首题复用回执验收卡设计

## 背景

首题复用回执模板已经把“读分数、带证据、认阻断、开下一题”转成可填写字段，但模板本身还没有判断回执质量。用户可能填写了动作，却没有明确证据、目标分或下一题入口，导致下一轮训练看似启动，实则没有把归档复用落到可验收结果。

## 目标

在“首题复用回执模板”和“下一轮训练建议”之间增加“首题复用回执验收卡”。验收卡把 4 个回执字段转成 4 条验收项：分数对齐、证据引用、阻断判断、下一题接续。所有内容继续由本地规则派生，不依赖后端、付费 AI 或第三方服务。

## 方案

首题复用回执验收卡由 `buildPracticeSessionFirstQuestionReuseReceipt` 派生：

- 回执模板为空时，显示等待验收状态。
- 回执模板处于回修状态时，显示“回修复用回执待验收”，强调是否补齐阻断修复证据。
- 回执模板处于 ready 状态时，显示“首题复用回执待验收”，强调是否允许进入下一轮训练。
- 每个验收项包含验收目标、通过信号、缺失风险和补救动作。
- 主行动复用回执模板入口，避免新增路由、存储或付费能力。

## 用户价值

相比只提供回执模板，验收卡让用户知道自己填写的回执是否足以支撑下一轮训练。它把免费模拟面试链路从“建议”和“填写”推进到“验收”和“放行”，减少盲目进入下一题带来的重复低效。

## 验收

- Markdown 导出包含“首题复用回执验收卡”，且位于“首题复用回执模板”之后、“下一轮训练建议”之前。
- 前端战报展示 4 个验收项：分数对齐、证据引用、阻断判断、下一题接续。
- 空会话显示等待状态，不出现 `undefined`。
- 聚焦测试、全量测试、构建和 `git diff --check` 通过。
